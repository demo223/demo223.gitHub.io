# Dockerfile自定义镜像

# 自定义镜像

镜像就是包含了应用程序、程序运行的系统函数库、运行配置等文件的文件包。构建镜像的过程其实就是把上述文件 按照顺序逐层打包。

# Dockerfile

Dockerfile就是一个文本文件，其中包含一个个的指令(Instruction)，用指令来说明要执行什么操作来构建镜像。将来 Docker可以根据Dockerfile帮我们构建镜像。常见指令如下：

指令 说明 示例
FROM 指定基础镜像 FROM centos:6
ENV 设置环境变量，可在后面指令使用 ENV key value
COPY 拷贝本地文件到镜像的指定目录 COPY ./mysql-5.7.rpm /tmp
RUN 执行 Linux 的 shell 命令 RUN yum install gcc
EXPOSE 指定容器运行时监听的端口，是给镜像使用者看的 EXPOSE 8080
ENTRYPOINT 镜像中应用的启动命令 ENTRYPOINT java -jar xx.jar

我们可以基于Ubuntu基础影响，利用Dockerfile描述镜像结构，也可以直接基于JDK为基础镜像，省略前面的步骤

# 基础镜像
FROM openjdk:11.0-jre-buster
# 设定时区
ENV TZ=Asia/Shanghai
RUN ln -snf /usr/share/zoneinfo/$TZ /etc/localtime && echo $TZ > /etc/timezone
# 拷贝jar包
COPY hm-service.jar /app.jar
# 入口
ENTRYPOINT ["java", "-jar", "/app.jar"]

解释：

  1. FROM openjdk:11.0-jre-buster：指定了该镜像基于 OpenJDK 11 的 JRE 版本，并基于 Debian "Buster" 发行版构建。
  2. ENV TZ=Asia/Shanghai：设置了环境变量 TZ 为 "Asia/Shanghai"，表示该容器将会运行在中国上海时区。
  3. RUN ln -snf /usr/share/zoneinfo/$TZ /etc/localtime && echo $TZ > /etc/timezone：运行该命令来设置容器中的时区为 $TZ 变量所指定的时区，以保证应用程序正确地处理时间相关的操作。
  4. COPY hm-service.jar /app.jar：将当前目录下名为 hm-service.jar 的文件拷贝到容器中的 /app.jar 路径下，注意这里将文件重命名为 app.jar。
  5. ENTRYPOINT ["java", "-jar", "/app.jar"]：定义了镜像的入口命令。当容器启动时，它将自动执行该命令。这里表示启动 Java 进程，传递 /app.jar 作为参数。

对于这里的Dockerfile文件的构建，将会构建一个基于OpenJDK 11的JRE版本的镜像，其中包含了应用程序的jar包，并且将默认使用java -jar命令启动应用程序。

# 构建镜像

当编写好了Dockerfile，可以利用下面命令来构建镜像

docker build -t myImage:1.0 .
1

命令 docker build -t myImage:1.0 . 的含义如下：

  1. docker build: 构建一个 Docker 镜像。
  2. -t myImage:1.0: 使用 -t 参数来指定镜像的名称和标签。这里的镜像名称为 "myImage"，标签为 "1.0"。
  3. .: 指定当前目录为构建上下文，也就是告诉 Docker 在当前目录中查找 Dockerfile 和其他需要构建到镜像中的文件。

因此，执行这个命令将会在当前目录下查找名为 Dockerfile 的文件，并根据其中的指令构建一个名为 hmall 的 Docker 镜像。

然后通过docker images即可查看到创建的镜像。

# 创建并启动容器

docker run -d --name hmall \
-p 8080:8080 \
--network my_network hmall
1
2
3
  • docker run: 运行一个 Docker 容器。
  • -d: 后台运行容器。
  • --name hmall: 指定容器的名称为 hmall。
  • -p 8080:8080: 将容器内部的 8080 端口映射到主机的 8080 端口。这样，可以通过浏览器访问本地的 8080 端口来访问容器内部的应用程序。
  • --network my_network: 指定容器连接到名为 my_network 的自定义网络中。
  • hmall: 后面跟着要运行的镜像的名称。

因此，执行该命令将会在名为 my_network 的自定义网络中启动一个名为 hmall 的后台容器，并将容器内部的 8080端口映射到主机的 8080端口。您可以通过浏览器访问 http://localhost:8080 来访问该容器内部的应用程序。

此时通过服务ip地址:8080/...即可成功请求。

# nginx容器配置

创建nginx文件夹，并添加html文件夹和nginx.conf配置文件，作为nginx容器挂载的目录

docker run -d \
  --name hmall_nginx \
  -p 18080:18080 \
  -p 18081:18081 \
  -v /demo/hmall/nginx/html:/usr/share/nginx/html \
  -v /demo/hmall/nginx/nginx.conf:/etc/nginx/nginx.conf \
  -v /demo/hmall/nginx/log:/var/log/nginx \
  --network my_network \
  nginx

注意这里的nginx.conf里面的location需要和容器位置位置(/usr/share/nginx/html)匹配，

worker_processes  1;

events {
    worker_connections  1024;
}

http {
    include       mime.types;
    default_type  application/json;

    sendfile        on;
    
    keepalive_timeout  65;

    server {
        listen       18080;
        # 指定前端项目所在的位置
        location / {
            root /usr/share/nginx/html/hmall-portal;
        }

        error_page   500 502 503 504  /50x.html;
        location = /50x.html {
            root   html;
        }
        location /api {
            rewrite /api/(.*)  /$1 break;
            proxy_pass http://hmall:8080;
        }
    }
    server {
        listen       18081;
        # 指定前端项目所在的位置
        location / {
            root /usr/share/nginx/html/hmall-admin;
        }

        error_page   500 502 503 504  /50x.html;
        location = /50x.html {
            root   html;
        }
        location /api {
            rewrite /api/(.*)  /$1 break;
            proxy_pass http://hmall:8080;
        }
    }
}

# mysql容器创建

在当前目录下创建了相应的 hmall/mysql/data、hmall/mysql/conf 和 hmall/mysql/init 目录，并且添加 MySQL 配置文件和初始化脚本。

docker run -d \
 --name hmall_mysql \
 -p 13306:13306 \
 -e TZ=Asia/Shanghai \
 -e MYSQL_ROOT_PASSWORD=123456 \
 -v /demo/hmall/mysql/data:/var/lib/mysql \
 -v /demo/hmall/mysql/conf:/etc/mysql/conf.d \
 -v /demo/hmall/mysql/init:/docker-entrypoint-initdb.d \
 --network my_network \ 
 mysql

  • -d：通过该参数指定容器以后台模式运行。
  • --name hmall_mysql：通过该参数指定容器的名称为 "hmall_mysql"。
  • -p 13306:13306：通过该参数将主机的 13306 端口映射到容器的 13306 端口，用于网络访问。
  • -e TZ=Asia/Shanghai：通过该参数设置容器的时区为亚洲/上海。
  • -e MYSQL_ROOT_PASSWORD=123456：通过该参数设置 MySQL root 用户的密码为 "123456"。请确保将此密码替换为更安全的密码。
  • -v ./hmall/mysql/data:/var/lib/mysql：通过该参数将主机上的目录 ./hmall/mysql/data 挂载到容器的 /var/lib/mysql 目录，用于持久化存储 MySQL 数据。
  • -v ./hmall/mysql/conf:/etc/mysql/conf.d：通过该参数将主机上的目录 ./hmall/mysql/conf 挂载到容器的 /etc/mysql/conf.d 目录，用于自定义 MySQL 配置。
  • -v ./hmall/mysql/init:/docker-entrypoint-initdb.d：通过该参数将主机上的目录 ./hmall/mysql/init 挂载到容器的 /docker-entrypoint-initdb.d 目录，用于初始化数据库。
