# Docker创建容器

# 镜像和容器

当我们利用Docker安装应用时，Docker会自动搜索并下载应用镜像（image）。镜像不仅包含应用本身，还包含应 用运行所需要的环境、配置、系统函数库。Docker会在运行镜像时创建一个==隔离环境==，称为容器（container）。 镜像仓库：存储和管理镜像的平台，Docker官方维护了一个公共仓库：DockerHub (opens new window)。

# 数据卷

数据卷（volume）是一个虚拟目录，是容器内目录与宿主机目录之间映射的桥梁。

为什么不让容器目录直接指向宿主机目录呢？

  • 安全性：直接将容器内的目录与宿主机目录进行映射可能存在安全风险。如果容器有恶意代码或意外操作修改了宿主机上的数据，可能会导致宿主机的数据被破坏或篡改。
  • 可移植性：使用数据卷可以使容器与宿主机的解耦，提供更好的可移植性。如果容器需要在不同的环境中部署，或者需要迁移到其他主机，使用数据卷可以方便地将数据迁移到新的环境中。
  • 备份和恢复：使用数据卷可以使数据的备份和恢复变得更加方便。通过备份数据卷，可以轻松地将数据从一个容器转移到另一个容器，或者在需要时还原数据。
  • 灵活性：使用数据卷可以实现多个容器之间的数据共享。通过映射相同的数据卷到多个容器，这些容器可以共享相同的数据，使得它们可以共同处理和访问相同的数据。

查看指定Docker容器的数据卷

docker inspect <容器ID或容器名称>
1

查看Docker中所有的数据卷

docker volume ls

# 创建MySQL容器

下面以Docker上安装MySQL8为例，创建mysql容器

安装命令：

docker run -d -i \
--name mysql \
-v /usr/local/server/mysql/datadir:/var/lib/mysql \
-v /usr/local/server/mysql/conf.d:/etc/mysql/conf.d \
-v /etc/localtime:/etc/localtime \
--privileged=true \
-p 3306:3306 \
-e MYSQL_ROOT_PASSWORD=123456 \
--restart=always \
mysql:8.0

  • -d: 在后台模式下运行容器。
  • -i: 以交互模式启动容器。
  • --name mysql: 指定容器的名称为 "mysql"。
  • -v /usr/local/server/mysql/datadir:/var/lib/mysql: 将宿主机上的 /usr/local/server/mysql/datadir 目录挂载到容器内的 /var/lib/mysql 目录，用于持久化存储 MySQL 数据。
  • -v /usr/local/server/mysql/conf.d:/etc/mysql/conf.d: 将宿主机上的 /usr/local/server/mysql/conf.d 目录挂载到容器内的 /etc/mysql/conf.d 目录，用于存放自定义的 MySQL 配置文件。
  • -v /etc/localtime:/etc/localtime: 将宿主机的时间配置文件挂载到容器内，保持容器内外时间一致。
  • --privileged=true: 给容器赋予特权，以便使用某些特权操作（不建议在生产环境中使用）。
  • -p 3306:3306: 将宿主机的 3306 端口映射到容器的 3306 端口，使得可以通过宿主机访问 MySQL 服务。
  • -e MYSQL_ROOT_PASSWORD=123456: 设置 MySQL 的 root 用户密码为 "123456"。
  • --restart=always: 当容器意外停止时，自动重启容器。
  • mysql:8.0: 使用名为 "mysql" 的 Docker 镜像，版本为 8.0。

docker run 命令中使用的镜像名称为 mysql:8.0。执行该命令时，Docker 首先会检查本地是否存在标记为 mysql:8.0 的镜像。如果本地没有该镜像，Docker 将会自动从 Docker Hub 下载该镜像并保存到本地。

所以，当执行 docker run 命令时，即使没有提前执行 docker pull mysql:8.0 命令，Docker 也会根据需要自动下载 mysql:8.0 镜像，并使用该镜像创建和运行名为 "mysql" 的容器。

修改密码:

#进入容器
docker exec -it mysql /bin/bash
#登录mysql
mysql -u root -p
#修改登录权限
ALTER USER 'root'@'%' IDENTIFIED WITH mysql_native_password BY '123456';

使用 exec 命令来在容器中执行指定的命令。参数解释如下：

  • -it: 将终端与容器进行交互连接。
  • mysql: 容器名称或容器 ID，根据您运行时所分配的名称或 ID 进行替换。
  • /bin/bash: 指定在容器中执行的命令为 Bash 终端。

docker cp /usr/share/zoneinfo/Asia/Shanghai mysql:/usr/share/zoneinfo/Asia

docker exec -it mysql bash

cp /usr/share/zoneinfo/Asia/Shanghai /etc/localtime

docker restart mysql

# Docker常见命令

  • docker run：创建并运行一个容器，处于运行状态

    docker run --name 容器名称 -p 80:80 -d 镜像名称
    1

  • docker pause：让一个运行的容器暂停

    docker pause 容器名称或容器id
    1

  • docker unpause：让一个容器从暂停状态恢复运行

    docker unpause 容器名称或容器id
    1

  • docker stop：停止一个运行的容器

    docker stop 容器名称或容器id
    1

  • docker start：让一个停止的容器再次运行

    docker start 容器名称或容器id
    1

  • docker rm：删除一个容器

    docker rm 容器名称或容器id
    1

    通常在删除镜像之前，需要先删除使用该镜像的所有容器。如果不这样做，可能会收到类似image is being used by stopped container 3733671c1d50 的错误信息。但是，如果你确实想要删除所有使用该镜像的容器，可以使用下面的命令来强制删除。

    docker rmi -f 镜像名称
    1
  • docker logs -f 容器(名称:tag或id) -f 表示可以持续查看日志
  • docker ps 查看容器状态
