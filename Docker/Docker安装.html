# Docker安装

# 一、准备工作

# 系统要求

Docker 支持 64 位版本 CentOS 7/8，并且要求内核版本不低于 3.10。 CentOS 7 满足最低内核的要求，但由于内核版本比较低，部分功能（如 overlay2 存储层驱动）无法使用，并且部分功能可能不太稳定。

# 卸载旧版本

旧版本的 Docker 称为 docker 或者 docker-engine，使用以下命令卸载旧版本：

yum remove docker \
docker-client \
docker-client-latest \
docker-common \
docker-latest \
docker-latest-logrotate \
docker-logrotate \
docker-selinux \
docker-engine-selinux \
docker-engine \
docker-ce

用于卸载 Docker 相关组件和软件包

  • docker-client：Docker 客户端是一个命令行工具，用于与 Docker 服务器进行交互，可以用来管理 Docker 容器、镜像和网络等资源。
  • docker-client-latest：这是 Docker 客户端的最新版本。
  • docker-common：这是 Docker 的一些共享文件和配置文件。
  • docker-latest：这是 Docker 的最新版本。
  • docker-latest-logrotate：此软件包包含了用于旧版本 Docker 日志轮转的配置文件。
  • docker-logrotate：此软件包包含了用于 Docker 日志轮转的配置文件。
  • docker-selinux：这是 Docker 的安全增强模块，用于提供 SELinux支持。
  • docker-engine-selinux：这是 Docker 引擎的 SELinux策略模块。
  • docker-engine：这是 Docker 引擎，用于执行容器化应用程序。
  • docker-ce：这是 Docker 社区版，是社区支持的免费版本。

通过执行 yum remove 命令加上上述软件包的名称，可以将它们从系统中移除。请注意，在使用该命令之前，应该确保你真的想要删除这些软件包，并且备份了相关数据，因为删除后将无法恢复。

# 二、使用 yum 安装

执行以下命令安装依赖包：

yum install -y yum-utils \
device-mapper-persistent-data \
lvm2 --skip-broken
1
2
3
  • yum-utils：yum 辅助工具，提供了一些有用的命令行工具，例如 yum-config-manager、package-cleanup 和 needs-restarting 等，使得 yum 的使用更加方便和高效。
  • device-mapper-persistent-data：设备映射持久化数据，是针对 LVM（逻辑卷管理器）设备映射的一个扩展组件，负责将逻辑卷和物理磁盘之间的映射关系保存到一个持久化数据库中，以便系统重启后可以恢复这些信息。
  • lvm2：逻辑卷管理器 2，是一个在 Linux 中管理逻辑卷的软件，可以动态地创建、调整、删除逻辑卷，从而实现更好的存储管理。

--skip-broken 选项表示如果有其他软件包的依赖项无法解决，则跳过这些软件包的安装。

鉴于国内网络问题，强烈建议使用国内源。

执行下面的命令添加 yum 软件源：

yum makecache fast

这些命令用于设置 Docker 镜像源为阿里云镜像源，并更新 yum 的缓存。

  1. yum-config-manager --add-repo https://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo：这个命令将添加一个 Docker 镜像源的配置文件到 /etc/yum.repos.d/ 目录下，使用的是阿里云镜像源的 URL。
  2. sed -i 's/download.docker.com/mirrors.aliyun.com\/docker-ce/g' /etc/yum.repos.d/docker-ce.repo：这个命令将修改 /etc/yum.repos.d/docker-ce.repo 文件中的镜像源地址，将原来的 download.docker.com 替换为阿里云镜像源 mirrors.aliyun.com/docker-ce。
  3. yum makecache fast：这个命令会重新生成 yum 的缓存，以便使用新的镜像源。

然后输入如下命令更新 yum 软件源缓存，并安装 docker-ce。：

yum install -y docker-ce

# 四、镜像加速器

国内从 Docker Hub 拉取镜像有时会遇到困难，此时可以配置镜像加速器。国内很多云服务商都提供了国内加速器服务，例如：

  • 阿里云加速器(点击管理控制台 -> 登录账号(淘宝账号) -> 右侧镜像工具 -> 镜像加速器 -> 复制加速器地址) (opens new window)
  • 网易云加速器 https://hub-mirror.c.163.com (opens new window)
  • 百度云加速器 https://mirror.baidubce.com (opens new window)

由于镜像服务可能出现宕机，建议同时配置多个镜像。

# 配置镜像

创建或修改 /etc/docker/daemon.json：

sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<-'EOF'
{
    "registry-mirrors": [
        "https://docker.m.daocloud.io",
        "https://dockerproxy.com",
        "https://docker.mirrors.ustc.edu.cn",
        "https://docker.nju.edu.cn"
    ]
}
EOF
sudo systemctl daemon-reload
sudo systemctl restart docker

具体命令如下：

  1. sudo mkdir -p /etc/docker：创建一个名为 /etc/docker 的目录，如果该目录不存在的话。这个目录用于存储 Docker 的配置文件。
  2. sudo tee /etc/docker/daemon.json <<-'EOF'：使用 tee 命令创建或覆盖 /etc/docker/daemon.json 文件，并将后续输入的内容写入该文件。<<-'EOF' 表示后续输入作为多行输入直到遇到 EOF 结束。
  3. 多行输入：配置多个镜像加速器的地址。
  4. EOF：结束多行输入。
  5. sudo systemctl daemon-reload：重新加载 systemd 的配置，以便使其知道 Docker 的配置文件已经被修改。
  6. sudo systemctl restart docker：重启 Docker 服务，使新的配置生效。

# 检查加速器是否生效

命令行执行 docker info，如果从结果的最下面看到了如下内容，说明配置成功。

 Registry Mirrors:
  https://docker.m.daocloud.io/
  https://dockerproxy.com/
  https://docker.mirrors.ustc.edu.cn/
  https://docker.nju.edu.cn/
