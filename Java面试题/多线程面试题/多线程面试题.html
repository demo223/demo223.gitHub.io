# 多线程面试题

# 一、并行和并发有什么区别？

  • 并行：多个任务在计算机中==同时执行==

  • 并发：多个任务在计算机中==交替执行==

并发：在一个CPU上串行执行

并行：多个CPU上同时执行任务

# 二、线程和进程的区别？

进程是==程序运行和资源分配==的基本单位，一个程序至少有一个进程，一个进程至少有一个线程，但一个进程一般有多个线程。

进程在运行过程中，需要拥有独立的内存单元，否则如果申请不到，就会挂起。而多个线程能共享内存资源，这样就能降低运行的门槛，从而效率更高。

线程是cpu调度和分派的基本单位，在实际开发过程中，一般是考虑多线程并发。

    public class Test {

	}
}

class MyThread extends Thread {

# 四、runnable 和 callable 有什么区别

  1. 返回值类型不同：Runnable 的 run 方法没有返回值，Callable 的 call 方法有一个泛型返回值。
  2. 抛出异常：Runnable 的 run 方法只能抛出==运行时异常==，==无法捕获处理==，而 Callable 的 call 方法可以抛出异常，可以获取异常信息。
  3. 使用场景不同：一般情况下，Runnable 用于执行没有返回值的异步操作，Callable 用于执行需要返回值的异步操作，并且需要处理可能会抛出的异常。

# 五、线程包括哪些状态，状态之间是如何变化的

Java线程在运行的生命周期中，在任意给定的时刻，只能处于下列6种状态之一：

  1. new ：初始状态，线程被创建，但是还==没有==调用start方法。

  2. runnable ：可运行状态，线程正在JVM中执行，但是有可能在等待操作系统的调度。

    1. 就绪（Runnable）：当调用 start() 方法后，线程进入就绪状态。此时，线程会加入到就绪队列中等待 CPU 时间片的分配。
    2. 运行（Running）：CPU 时间片被分配到该线程时，线程开始执行，并进入运行状态

  3. blocked ：阻塞状态，线程正在等待获取监视器锁。

  4. waiting ：等待状态，线程正在等待其他线程的通知或中断。

  5. timed_waiting ：超时等待状态，在waiting 的基础上增加了超时时间，即超出时间自动返回。

  6. terminated：终止状态，线程已经执行完毕。

线程的状态之间可以通过以下方式变化：

  1. 新建线程通过start()方法开始运行，变成可运行状态；
  2. 可运行线程会竞争CPU的时间片来执行，如果被其他线程抢占了CPU时间，它就会变成阻塞状态；
  3. 阻塞状态的线程如果获得了它需要的锁或者其他阻塞条件得到满足，就会变成可运行状态，竞争CPU的时间片；
  4. 等待状态的线程如果收到了唤醒信号，就会进入可运行状态，竞争CPU的时间片；
  5. 计时等待状态的线程如果等待时间到了，就会进入可运行状态，竞争CPU的时间片；
  6. 线程完成了它的工作或者出现了未捕获的异常，就会变成终止状态。

线程在创建之后默认为初始状态，在调用start方法之后进入可运行状态，可运行状态不代表线程正在运行，它有可能正在等待操作系统的调度。进入等待状态的线程需要其他线程的通知才能返回到可运行状态，而超时等待状态相当于在等待状态的基础上增加了超时限制，除了他线程的唤醒，在超时时间到达时也会返回运行状态。此外，线程在执行同步方法时，在没有获取到锁的情况下，会进入到阻塞状态。线程在执行完run方法之后，会进入到终止状态。

# 六、在 java 中 wait 和 sleep 方法的不同？

相同点：两者都可以暂停线程，在等待的过程中都可以被唤醒

不同点：

  1. 来自不同的类

    wait()来自 Object 类； sleep()来自 Thread 类(Thread的静态方法)

  2. 关于锁的释放

    wait()在等待的过程中==会释放锁==； sleep()在等待的过程中==不会释放锁==

    wait() 方法在等待的过程中会释放锁，是因为它被设计为在多线程之间同步对象状态。当一个线程调用了某个对象的 wait() 方法时，它会进入该对象的等待队列，并且释放对象的锁，此时其他线程可以获得该锁并继续执行。当有其他线程调用该对象的 notify() 或 notifyAll() 方法时，等待线程才会被唤醒继续执行。

    这种机制的存在是为了确保多个线程之间的协作和同步。如果 wait() 方法不释放锁，那么其他线程将无法访问该对象，从而导致死锁或其他并发问题。

    相反，sleep() 方法在等待的过程中不会释放锁，是因为它主要是为了暂停当前线程的执行，在指定的时间内不进行任何操作。如果线程在等待期间释放了锁，那么其他线程可能会在此期间修改对象状态，从而导致数据一致性问题。因此，sleep() 方法通常用于实现等待固定时间的延迟操作，而不是用于多线程之间的同步。如果需要在多个线程之间同步状态，应该使用 wait() 和 notify() 方法来实现。

  3. 使用的范围

    wait()必须在==同步代码块或同步方法==中使用； sleep()可以在==任何地方==使用

    是否需要捕获异常

    wait()==不需要==捕获InterruptedException异常； sleep()==需要==捕获InterruptedException异常；

    这样的回答是错误的，无论是 wait() 还是 sleep() 方法，都需要捕获 InterruptedException 异常，以便正确地处理线程中断。

# 七、新建 T1、T2、T3 三个线程，如何保证它们按顺序执行？

可以使用线程的join()方法来保证线程的执行顺序。==join()方法的作用是等待该线程终止，然后继续执行当前线程==。具体实现如下：

  1. 在T1线程中，先执行完T1线程的任务，然后调用T2线程的join()方法，让T2线程先执行，等T2线程执行完毕后，T1线程再继续执行自己的任务。
  2. 在T2线程中，先执行完T2线程的任务，然后调用T3线程的join()方法，让T3线程先执行，等T3线程执行完毕后，T2线程再继续执行自己的任务。
  3. 在T3线程中，执行完T3线程的任务后，所有线程的任务就全部完成了。

下面是一个示例代码：

public class Test1 {
	public static void main(String[] args) throws InterruptedException {
		Thread T1 = new Thread(() -> {
			// T1线程任务
			System.out.println("T1");
		});

		Thread T2 = new Thread(() -> {
			// T2线程任务
			System.out.println("T2");
		});

		Thread T3 = new Thread(() -> {
			// T3线程任务
			System.out.println("T3");
		});

		T1.start();
		T1.join();
		// 加入线程t1,只有t1线程执行完毕以后，再次执行该线程
		T2.start();
		T2.join();
		// 加入线程t2,只有t2线程执行完毕以后，再次执行该线程
		T3.start();
		T3.join();
	}
}

# 八、线程的 run()和 start()有什么区别？

  1. 启动线程的方式不同: start() 方法可以异步地启动新线程来执行 run() 方法中的代码。而直接调用 run() 方法相当于同步地执行代码，因为当前线程会等待 run() 方法中的代码执行完毕之后才会继续运行。
  2. 对其它线程的影响不同: 调用start()会新建一个线程，并执行run()方法，而不会影响到主线程的运行，而直接调用run()方法会导致当前线程阻塞，其它线程也会被阻塞。
  3. 多次调用的效果不同: start()方法只能被调用一次，多次调用会抛出IllegalThreadStateException异常。而run()方法可以被调用多次，每次调用都会在当前线程中顺序执行该方法。

总结：

  1. start() 可以(异步)启动一个新线程，run()不能
  2. start()不能被重复调用，run()可以
  3. start()中的run()代码可以不执行完就继续执行下面的代码，即进行了线程切换。直接调用run()方法必须等待其代码全部执行完才能继续执行下面的代码。
  4. start() 实现了多线程，run()没有实现多线程。

单独调用 run()方法，是同步执行； 通过 start()调用 run()，是异步执行

调用 start()方法方可启动线程，而run()方法只是thread的一个普通方法调用，还是在主线程里执行。

# 九、如何停止一个正在运行的线程？

停止一个正在运行的线程是一件比较棘手的问题，因为线程的停止需要协作和同步，不能直接强制终止线程(使用stop方法强行终止)，否则会带来一些潜在的问题，例如线程资源未能正确释放，共享数据状态异常等。

通常可以通过以下两种方式停止一个正在运行的线程：

  1. 使用退出标志

    定义一个退出标志，也就是一个共享变量，当共享变量满足一定的条件时，线程终止自己的运行。在线程中，可以定时检查共享变量的状态，如果共享变量满足终止条件，则退出线程的循环，从而终止线程的运行。

    示例代码：

    public class MyThread extends Thread {
	volatile boolean flag = false;     // 线程执行的退出标记
	public static void main(String[] args) throws InterruptedException {
		// 创建MyThread对象
		MyThread t1 = new MyThread();
		t1.start();
		// 主线程休眠6秒
		Thread.sleep(6000);
		// 更改标记为true
		t1.flag = true;
	}

	@Override
	public void run() {
		while (!flag) {
			System.out.println("MyThread...run...");
			try {
				Thread.sleep(3000);
			} catch (InterruptedException e) {
				e.printStackTrace();
			}
		}
	}
}

    需要注意的是，==由于 Java 线程模型的特殊性，直接停止线程并不是一种安全的做法。因此，建议仅在必要情况下才使用这种方法，并且尽可能让线程自行结束==。如果需要停止线程，应该使用一种更加安全和可控的方式，例如设置标志位或者调用线程的 interrupt() 方法来触发异常并终止线程的执行。

    调用线程的interrupt()方法可以中断正在运行的线程。在线程中，需要定时检查线程是否被中断，如果线程被中断，则退出线程的循环，从而终止线程的运行。

    public class MyThread extends Thread {
    public void run() {
        while (!isInterrupted()) {
        
            // 线程执行的任务
        }
    }
}

// 调用线程的interrupt()方法停止线程
MyThread thread = new MyThread();
thread.start();
thread.interrupt();

# 十、notify()和 notifyAll()有什么区别？

notifyAll()：唤醒所有wait的线程

notify()：只随机唤醒一个 wait 线程

在 Java 多线程编程中，notify() 和 notifyAll() 都是 Object 类中的方法，用于唤醒处于等待状态的线程，让它们重新进入就绪状态，等待获取锁并继续执行。它们的主要区别在于：

  1. notify() 方法只会随机地唤醒等待队列中的一个线程，而 notifyAll() 方法会唤醒等待队列中的所有线程。
  2. notify() 方法只会将等待队列中的一个线程从等待状态转换为就绪状态，但是哪一个线程会被唤醒是不确定的，可能会导致某些线程永远无法被唤醒，从而导致死锁。而 notifyAll()方法会将所有等待队列中的线程全部唤醒，确保所有线程都有机会获取锁并执行。

因此，一般来说，当多个线程等待同一个条件变量时，使用 notifyAll() 方法更安全可靠，可以避免因唤醒不确定性而导致的死锁问题。而如果只需要唤醒一个线程，且能够确保被唤醒的线程能够满足条件并顺利执行，使用 notify() 方法可以提高系统的效率。

需要注意的是，当多个线程共享同一个对象时，如果使用 notify() 方法可能会导致死锁，因为只有随机唤醒一个等待线程，如果该线程还没有释放对象锁，那么其他等待线程将会一直处于等待状态。因此，在大多数情况下，应该优先使用 notifyAll() 方法来确保所有等待线程都能够被唤醒。
