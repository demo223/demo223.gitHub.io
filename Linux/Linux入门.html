# Linux系统安装jdk

finalShell的上传按钮将指定的jdk文件上传到服务器

tar -zxvf jdk-8u171-linux-x64.tar.gz -C /usr/local
1

注意：是usr不是user

将jdk解压到我们上面创建的指定的文件夹下面

然后配置环境变量，需要先进入到 /etc/profile下面

vim /etc/profile
1

然后键入G字母直接到达文件末尾

然后进入插入模式，将jdk的环境变量配置到这里

JAVA_HOME=/usr/local/jdk1.8.0_171
PATH=$JAVA_HOME/bin:$PATH
1
2

然后esc退出插入模式，键入:wq保存退出

然后在使用如下命令重新加载配置

source /etc/profile
1

然后输入java -version测试是否配置好了

# Linux上面安装Tomcat

# 防火墙操作

  • 开启防火墙 systemctl start firewalld
  • 查看防火墙状态 systemctl status firewalld 或者使用firewall-cmd --state
  • 暂时关闭防火墙 systemctl stop firewalld
  • 永久关闭防火墙 systemctl disable firewalld

# 安装Tomcat并开启服务

操作步骤：

  1. 使用FinalShell自带的上传工具将Tomcat的二进制发布包上传到Linux

  2. 解压安装包，命令为 tar -zxvf apache-tomcat-7.0.57.tar.gz -C /usr/local

  3. 进入Tomcat的bin目录启动服务，命令为 ./startup

    cd /usr/local/apache-tomcat-7.0.57/bin
    1
    sh startup.sh
    1

  4. 查看启动日志

    more /usr/local/apache-tomcat-7.0.57/logs/catalina.out
    1

    或者使用

    tail -f /usr/local/apache-tomcat-7.0.57/logs/catalina.out
    1

    如果没有报错信息就说明启动成功

  5. 还可以通过查看进程的方式查看Tomcat是否启动成功

    ps -ef | grep tomcat
    1

  6. 在防火墙开启指定端口

    firewall-cmd --zone=public --add-port=8080/tcp --permanent
    1

    然后重启防火墙

    firewall-cmd --reload
    1

    可以通过如下命令来查看防火墙的开放端口

    firewall-cmd --permanent --zone=public --list-ports
    1

  7. 接下来可以通过外网进行访问8080端口了

    如果无法访问需要开放云服务器的8080端口

    不能只开启80端口，必须开启8080端口

    8080端口是被用于WWW代劳服务，要是云服务器安装了Apache Tomcat web server后，默认的apache-tomcat就是8080号端口。

# 关闭Tomcat服务

停止Tomcat服务的方式：

  1. 运行Tomcat的bin目录中提供的停止服务的脚本文件

    sh shutdown.sh
./shutdown.sh
    1
    2

  2. 结束Tomcat进程

    • 先查看Tomcat进程，获得进程id
    • 再执行命令结束进程 kill -9 pid

    注意：kill命令是Linux提供的用于结束进程的命令，-9表示强制结束

    查看Tomcat进程

    ps -ef | grep tomcat
    1

    在查出来Tomcat的进程后，结束进程

    kill -9 进程id
    1

正常情况还是建议使用第一种方式，在无法关闭的时候才使用第2种

RPM（Red-Hat Package Manager）RPM软件包管理器，是红帽Linux用于管理和安装软件的工具

# 登录MySQL

登录MySQL数据库，查阅临时密码

  • cat /var/log/mysqld.log 查看文件内容

  • cat /var/log/mysqld.log | grep password 查看文件内容中包含password的行信息(空格不要)

登录mysql，输入临时密码(密码输入默认不显示)

mysql -uroot -p
1

修改密码

set global validate_password_length=4; #设置密码长度最低位数 set global validate_password_policy=LOW; #设置密码安全等级低，便于密码修改成root set password = password('root'); #设置密码为root

grant all on . to 'root'@'%' identified by 'root'; #开启访问权限

上面如果不起作用试一下 GRANT ALL PRIVILEGES ON . TO 'root'@'%' IDENTIFIED BY 'root密码';

flush privileges; # 刷新

exit; # 退出

在使用得到的最新账号进行登录即可

    1

  4. 运行

    java -jar helloworld-1.0-SNAPSHOT.jar
    1

  5. 浏览器访问

手工部署项目存在的问题

  • 线上程序不会采用控制台霸屏的形式运行程序，而是将程序在后台运行
  • 线上程序不会将日志输出到控制台，而是输出到日志文件，方便运维查阅信息

nohup 命令：英文全称 no hang up（不挂起），用于不挂断地运行指定命令，退出终端不会影响程序的运行 语法格式： nohup Command [ Arg … ] [&]

参数说明： Command：要执行的命令

Arg：一些参数，可以指定输出文件

&：让命令在后台运行

举例： nohup java -jar boot工程.jar &> hello.log & #后台运行java -jar命令，并将日志输出到hello.log文件

    #!/bin/sh
echo =================================
echo  自动化部署脚本启动
echo =================================

echo 停止原来运行中的工程
APP_NAME=helloworld

tpid=`ps -ef|grep $APP_NAME|grep -v grep|grep -v kill|awk '{print $2}'`
if [ ${tpid} ]; then
    echo 'Stop Process...'
    kill -15 $tpid
fi
sleep 2
tpid=`ps -ef|grep $APP_NAME|grep -v grep|grep -v kill|awk '{print $2}'`
if [ ${tpid} ]; then
    echo 'Kill Process!'
    kill -9 $tpid
else
    echo 'Stop Success!'

echo 准备从Git仓库拉取最新代码

git pull 
echo 代码拉取完成

output=`mvn clean package -Dmaven.test.skip=true`

cd target

echo 启动项目

echo 项目启动完成
