# Mybatis-plus

# 一、快速入门

# 1.1 引入依赖

MybatisPlus提供了starter，实现了自动Mybatis以及MybatisPlus的自动装配功能，依赖如下：

<dependency>
    <groupId>com.baomidou</groupId>
    <artifactId>mybatis-plus-boot-starter</artifactId>
    <version>3.5.3.1</version>
</dependency>

# 二、常见注解

MybatisPlus是根据实体类的信息来推断出表的信息，从而生成SQL的。默认情况下：

  • MybatisPlus会把实体的类名驼峰转下划线作为表名
  • MybatisPlus会把实体的所有变量名驼峰转下划线作为表的字段名，并根据变量类型推断字段类型
  • MybatisPlus会把名为id的字段作为主键

但很多情况下，默认的实现与实际场景不符，因此MybatisPlus提供了一些注解便于我们声明表信息。

# @TableName

  • 描述：表名注解，标识实体类对应的表

  • 使用位置：实体类

示例：

@TableName("t_user")
public class User implements Serializable {

}
1
2
3
4

TableName注解除了指定表名以外，还可以指定很多其它属性：

属性 类型 默认值 必须指定 描述
value String "" 否 表名
schema String "" 否 schema
keepGlobalPrefix boolean false 否 是否保持使用全局的 tablePrefix 的值
（当全局 tablePrefix 生效时）
resultMap String "" 否 xml 中 resultMap 的 id
（用于满足特定类型的实体类对象绑定）
autoResultMap boolean false 否 是否自动构建 resultMap 并使用
（如果设置 resultMap 则不会进行
resultMap 的自动构建与注入）
excludeProperty String[] {} 否 需要排除的属性名 @since 3.3.1

# @TableId

  • 描述：主键注解，标识实体类中的主键字段
  • 使用位置：实体类的主键字段

示例：

@TableName("t_user")
public class User implements Serializable {

    private static final long serialVersionUID = 1L;

    @TableId(value = "id", type = IdType.ASSIGN_ID)
    private Long id;
}

TableId注解支持两个属性：

属性 类型 必须指定 默认值 描述
value String 否 "" 表名
type Enum 否 IdType.NONE 指定主键类型

IdType支持的类型有：

值 描述
AUTO 数据库 ID 自增，该类型请确保数据库设置了 ID自增 否则无效
NONE 无状态，该类型为未设置主键类型（注解里等于跟随全局，全局里约等于 INPUT）
INPUT insert 前自行 set 主键值
ASSIGN_ID 分配 ID(主键类型为 Number(Long 和 Integer)或 String)(since 3.3.0),使用接口IdentifierGenerator的方法nextId(默认实现类为DefaultIdentifierGenerator雪花算法)
ASSIGN_UUID 分配 UUID,主键类型为 String(since 3.3.0),使用接口IdentifierGenerator的方法nextUUID(默认 default 方法)
ID_WORKER 分布式全局唯一 ID 长整型类型(please use ASSIGN_ID)
UUID 32 位 UUID 字符串(please use ASSIGN_UUID)
ID_WORKER_STR 分布式全局唯一 ID 字符串类型(please use ASSIGN_ID)

这里比较常见的有三种：

  • AUTO：利用数据库的id自增长
  • INPUT：手动生成id
  • ASSIGN_ID：雪花算法生成Long类型的全局唯一id，这是默认的ID策略

# @TableField

描述：普通字段注解

示例：

@TableName("user")
public class User {
    @TableId
    private Long id;
    private String name;
    private Integer age;
    @TableField("isMarried")
    private Boolean isMarried;
    @TableField("`concat`")
    private String concat;
}

一般情况下我们并不需要给字段添加@TableField注解，一些特殊情况除外：

  • 成员变量名与数据库字段名不一致

  • 成员变量是以isXXX命名，按照JavaBean的规范，MybatisPlus识别字段时会把is去除，这就导致与数据库不符。

  • 成员变量名与数据库一致，但是与数据库的关键字冲突。使用@TableField注解给字段名添加``转义，例如

    order就是数据库的关键字。

常用功能：

功能1：比如说数据库中字段为create_time,而实体类的属性为createTime，前提是在全局策略配置中将驼峰命名关闭。一般是这个不需要我们自己转，全局里面会配置驼峰命名字段转换。

@TableField(value = "create_time")
private Date createTime;
1
2

功能2：又比如在实体类中有一个属性为updateTime，但是在数据库中没有这个字段，

但是在执行插入操作时给实体类的updateTime属性赋值了，那么可以通过在实体类的

updateTime属性上添加@TableField注解

@TableField(exist = false)
private Date updateTime;
1
2

执行代码的时候如果实体类中有数据库中不存在的字段就会报错，这个时候必须加上@TableField(exist = false)，这样就不会报错了。

例如查询所有的时候底层代码会默认执行

select  name,age, ... from tb_user;
1

会将表中所有的字段列举出来，而不是使用select *

功能3：在做日常数据操作时，还是有很多重复的操作

  • 新增时需要填充创建日期，修改日期
  • 修改时需要填充修改日期

【第一步】给pojo类中给需要自动填充的属性加上@TableField(fill = FieldFill.XXX)注解

// @TableField(value = "create_time")   //  设置数据库与实体类之间的映射
@TableField(fill = FieldFill.INSERT)    //  设置在插入数据的时候创建日期
private Date createTime;
// @TableField(exist = false)  //  表示该属性不是数据库中的字段
@TableField(fill = FieldFill.INSERT_UPDATE) //  设置在插入和修改数据的时候创建日期
private Date updateTime;

【第二步】自定义数据处理器MyMetaObjectHandler实现MetaObjectHandler接口。

@Component
@Slf4j
public class MyMetaObjectHandler implements MetaObjectHandler {
	@Override
	public void insertFill(MetaObject metaObject) {
		log.info("添加数据:公共字段自动填充");
		// setFieldValByName底层还是调用的metaObject.setValue，只是加了一些判断前提条件
		// metaObject.setValue("createTime", LocalDate.now());
		setFieldValByName("createTime", LocalDateTime.now(),metaObject);
		setFieldValByName("updateTime", LocalDateTime.now(),metaObject);
		// 从当前线程中获取用户信息
		Employee employee = ThreadContextUtil.getEmployee();
		setFieldValByName("createUser",employee.getId(),metaObject);
		setFieldValByName("updateUser",employee.getId(),metaObject);
	}


	@Override
	public void updateFill(MetaObject metaObject) {
		log.info("修改数据:公共字段自动填充");
		setFieldValByName("updateTime",LocalDateTime.now(),metaObject);
		// 从当前线程中获取用户信息
		Employee employee = ThreadContextUtil.getEmployee();
		setFieldValByName("updateUser",employee.getId(),metaObject);
	}
}

# 三、常见配置

MybatisPlus也支持基于yaml文件的自定义配置，详见官方文档： 使用配置 | MyBatis-Plus (opens new window)

大多数的配置都有默认值，因此我们都无需配置。但还有一些是没有默认值的，例如:

  • 实体类的别名扫描包
  • 全局id类型
mybatis-plus:
  type-aliases-package: com.itheima.mp.domain.po
  global-config:
    db-config:
      id-type: auto # 全局id类型为自增长

需要注意的是，MyBatisPlus也支持手写SQL的，而mapper文件的读取地址可以自己配置：

mybatis-plus:
  mapper-locations: "classpath*:/mapper/**/*.xml" # Mapper.xml文件地址，当前这个是默认值。
1
2

可以看到默认值是classpath*:/mapper/**/*.xml，也就是说我们只要把mapper.xml文件放置这个目录下就一定会被加载。
