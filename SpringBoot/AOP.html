# 一、AOP的介绍

AOP为Aspect Oriented Programming的缩写，即面向切面编程（也叫面向方面），是一种可以通过预编译方式和运行期动态代理实现在不修改源代码的情况下给程序动态统一添加功能的一种技术。

编程思想 描述
面向对象编程(OOP) 简化代码：把重复代码纵向抽取到父类，OOP的三大特征继承、封装、多态，
它主要是为了实现编程的重用性、灵活性和扩展性，强调的是类之间的层次关系
面向接口编程 解耦：不同组件之间解耦，即使一个组件改变了也不会影响其它的
面向切面编程(AOP) 简化代码：把方法中的重复代码横向抽取到切面中，它是对方法的增强

OOP与AOP对比

①相同点

  • 都可以简化代码

②不同点

  • 面向对象：纵向抽取
  • 面向切面：横向抽取

# AOP问题导入

例如这里有两个类，两个类的方法1中的内容是重复的，而方法2是不重复的，此时我们就可以将方法1中的代码抽取出来放入父类中，这是典型的面向对象编程思想。

但是如果两个类的方法1中的内容只有部分是相同的，但是你又想将相同的方法抽取出来，这时再使用面向对象编程就无法实现了，因为相同的代码是存在于方法的内部，此时就可以使用AOP来实现了。

例如，有如下需求计算每个方法的执行时间

// 方法1
public void getUser(){
    long start = System.nanoTime();
   // 执行getUser事件
    long end = System.nanoTime();
    System.out.println(end - start);
}

// 方法2
public void updateUser(){
    long start = System.nanoTime();
   // 执行updateUser事件
    long end = System.nanoTime();
    System.out.println(end - start);
}

每个方法中我们看出有相同的内容，就是计算方法前后的时间差，也有不同的内容，方法不同。

如果每个方法上面都加上这个重复的代码，虽然可以解决上述问题，但是会存在很多问题

这样附加功能后，代码存在明显的问题：

  • 代码会变得非常臃肿
  • 核心逻辑的代码和非核心逻辑的代码混杂在一起，不利于开发和维护
  • 将来不管是核心代码还是非核心代码想要升级或调试bug，都非常不便

而且这种情况也不能将相同的代码抽取出来放在父类中，但是可以用到AOP思想来解决。

# AOP的实现原理

AOP的实现原理：基于动态代理完成

动态代理是23种设计模式中的一种，属于结构型模式。它的作用就是通过提供一个代理类，让我们在调用目标方法的时候，不再是直接对目标方法进行调用，而是通过代理类间接调用。让不属于目标方法核心逻辑的代码从目标方法中剥离出来。调用目标方法时先调用代理对象的方法，减少对目标方法的调用和打扰。

动态代理有两种实现方式：

  • JDK动态代理：需要被代理的目标类必须实现相应接口。利用反射机制生成一个实现代理接口的匿名类，在调用具体方法前调用InvokeHandler来处理。
  • CGlib动态代理：通过继承被代理的目标类实现代理，所以不需要目标类实现接口。利用ASM（开源的Java字节码编辑库，操作字节码）开源包，将代理对象类的class文件加载进来，通过修改其字节码生成子类来处理。

二者区别：JDK代理只能对实现接口的类生成代理；CGlib是针对类实现代理，对指定的类生成一个子类，并覆盖其中的方法，这种通过继承类的实现方式，不能代理final修饰的类。

关于AOP中使用的是JDK代理还是CGLib代理

SpringBoot默认配置spring.aop.proxy-target-class=true，此时无论目标是否实现接口，都是采用cglib

如果设置了 spring.aop.proxy-target-class=false，那么又分两种情况

  • 如果在AOP中被代理的是一个接口的实现类，那么AOP中使用就是JDK动态代理
  • 如果在AOP中被代理的类是一个普通类，那么AOP中使用就是CGlib动态代理

# AOP的相关术语

AOP相关术语 描述
切面 切面由切入点(Pointcut)和通知(Advice)组成，它既包括了横切逻辑的定义、
也包括了连接点的定义，所以通常来说它是一个类，就是被@Aspect注解注上的类。
通知/增强 AOP在特定的切入点上执行的增强处理，也就是具体的横切逻辑。简单来说就是
定义了是干什么的，具体是在哪干。Spring AOP提供了5种Advice类型给我们：
前置、后置、返回、异常、环绕，下面会详细介绍
连接点 目标类中每个成员方法都可以称之为连接点，一个连接点总是代表一个方法执行
可以这么理解：把方法排成一排，每一个横切位置看成x轴方向，把方法从上到下
执行的顺序看成y轴，x轴和y轴的交叉点就是连接点。
切入点 定位连接点的方式。上面也说了，每个方法都可以称之为连接点，我们
具体定位到某一个方法就称为切入点，在程序中主要体现为书写切入点表达式。
切点分为execution方式和annotation方式。前者可以用路径表达式指定哪些类织入切面，后者可以指定被哪些注解修饰的代码织入切面。
织入 就是通过动态代理，将通知添加到目标类的具体连接点上的过程。
目标 表示被代理的目标对象。
代理 表示向目标对象应用通知之后创建的代理对象。

# AspectJ相关的注解

@Aspect：标记这个类是一个切面类

AspectJ增强相关注解

注解 描述
@Pointcut 表示定义重用切入点表达式，一次定义，处处使用，一处修改，处处生效
@Around 表示将当前方法标记为环绕通知
@Before 表示将当前方法标记为前置通知
@AfterReturning 表示将当前方法标记为返回通知
@AfterThrowing 表示将当前方法标记为异常通知
@After 表示将当前方法标记为后置通知

  1. 第一步：编写切面类

    @Component
@Aspect //  @Aspect注解标记这个类是一个切面类
@Slf4j	//	日志打印
public class Aspect1 {
   @SneakyThrows
   // 声明切入点
   @Around("execution(public * com.zq.service.impl.UserServiceImpl.save(User User))")
   // 定义一个通知方法
   public Object around(ProceedingJoinPoint pjp){
      long start = System.nanoTime();
      //  运行切入点的方法
      Object res = pjp.proceed();
      long end = System.nanoTime();
      log.info("AOP1执行时间为{}",end-start);
      return res;
   }
}

# 切点表达式

切点表达式用来匹配哪些方法需要被增强。切点表达式有两种写法：

方式1：execution 关键字

方式2：自定义注解方式

在上面的例子中，切入点表达式execution(public * com.zq.service.impl.UserServiceImpl.save(User User))是写死的，如果有很多地方要切入的话，就要在切面类中编写大量重复性的代码，扩展性和实用性不高，所以下面来学习一下更加强大的切入点表达式。

注意：AspectJ切入点表达式语法：

execution(<访问修饰符>? <返回类型> <全限定名>? <方法名称>(<参数类型>) <异常>?)
execution (public 返回值类型 包名.类名.方法名(参数类型 参数)
1
2

通过execution函数，可以定义切入的方法。代码块中带?符号的匹配式都是可选的，对于execution必不可少的只有三个：

  • 返回类型
  • 方法名
  • 参数

完整的传统切入点表达式：execution(public * com.zq.service.impl.UserServiceImpl.*(..))

上面最大可以简写为：execution(* *..*.*(..)) 表示匹配任意修饰符，返回值，包，类，方法，参数。

  • 用*号代替“权限修饰符”和“返回值”部分，表示“权限修饰符”和“返回值”不限，即任意类型，注意：这里一个*代表两部分，下面有介绍
  • 在包名的部分，使用*表示包名任意
  • 在包名的部分，使用*..表示包名任意、包的层次深度任意
  • 在类名的部分，使用*号表示类名任意，也可以可以使用*号代替类名的一部分，例如：
*Service
1

上面例子*Service表示匹配所有类名、接口名以Service结尾的类或接口（*号位置不限）

  • 在方法名部分，使用*号表示方法名任意，也可以使用*号代替方法名的一部分，例如：
*User
1

上面例子*User表示匹配所有方法名以User结尾的方法（*号位置不限）

  • 在方法参数列表部分，使用(..)表示参数列表任意
  • 在方法参数列表部分，使用(int,..)表示参数列表以一个int类型的参数开头，后面的任意
  • 在方法返回值部分，如果想要明确指定一个返回值类型，那么必须同时写明权限修饰符
  • 对于execution()表达式整体可以使用三个逻辑运算符号（了解，几乎不用）
    • execution() || execution()表示满足两个execution()中的任何一个即可
    • execution() && execution()表示两个execution()表达式必须都满足
    • !execution()表示不满足表达式的其他方法

# 切点表达式的重复使用

这里需要用到@Pointcut注解。在一处声明切入点表达式之后，在其它有需要的地方引用这个切入点表达式就好。易于维护，一处修改，处处生效。声明方式如下：

@Pointcut("execution(public * com.zq.service.impl.PersonServiceImpl.save(..))")
public void pt1() {
}

@Pointcut("execution(public * com.zq.service.impl.PersonServiceImpl.findAll(..))")
public void pt2() {
}

//	在同一个类中引用切点表达式
@Around("pt1() || pt2()")	//	满足其中任意一个表达式即可
@SneakyThrows
public Object around(ProceedingJoinPoint pjp) {
   long start = System.nanoTime();

   Object res = pjp.proceed();

   long end = System.nanoTime();
   log.info("AOP2执行时间为{}", end - start);
   return res;
}

# 5种通知类型

通知/增强（Advice）中提供的五中类型（非常重要）：

通知类型 描述
前置通知(@Before) 在我们执行目标方法之前运行
返回通知(@AfterReturning) 在我们的目标方法正常返回值后运行

后置通知(@After) 在我们目标方法运行结束之后 ，不管有没有异常
环绕通知(@Around) 功能最为强大的通知

通知的执行顺序可以根据下面的代码模型判断

# 通知中获取连接点

连接点就是目标方法，在Spring 中用 JoinPoint 对象抽象了连接点，用它可以获得方法执行时的相关信息，如方法 名、方法参数类型、方法实际参数等等。所有的通知方法上都可以添加连接点参数，但是必须是在第一个。

  • 对于 @Around 通知，获取连接点信息只能使用 ProceedingJoinPoint
  • 对于其他四种通知，获取连接点信息只能使用 JoinPoint，它是 ProceedingJoinPoint 的父类
@Slf4j
@Component
@Aspect
public class Aspect5 {
    //	环绕通知通过ProceedingJoinPoint获取连接点
    @Around("execution(* com.itheima.service.impl.PersonServiceImpl.*(..))")
    public Object around(ProceedingJoinPoint pjp) throws Throwable {
        //1. 获取方法签名
        MethodSignature signature = (MethodSignature) pjp.getSignature();
        //2. 获取方法对象Method
        Method method = signature.getMethod();
        log.info("方法对象:{}",method);
        //3. 获取方法的返回值类型
        Class returnType = signature.getReturnType();
        log.info("方法的返回值类型为{}",returnType);
        //4. 获取参数类型
        Class[] parameterTypes = signature.getParameterTypes();
        log.info("方法的参数类型为{}",Arrays.toString(parameterTypes));
        //5. 获取参数值
        Object[] args = pjp.getArgs();
        log.info("方法的参数值为{}",Arrays.toString(args));
        return pjp.proceed();
    }
    
    //	其他通知通过JoinPoint获取连接点
    @Before("execution(* com.itheima.service.impl.PersonServiceImpl.*(..))")
    public void before(JoinPoint jp) {
        //1. 获取方法签名
        MethodSignature signature = (MethodSignature) jp.getSignature();
        log.info("方法签名:{}",signature);
        //2. 获取方法对象Method
        Method method = signature.getMethod();
        log.info("方法对象:{}",method);
        //3. 获取方法的返回值类型
        Class returnType = signature.getReturnType();
        log.info("方法返回值类型:{}",returnType);
        //4. 获取参数类型
        Class[] parameterTypes = signature.getParameterTypes();
        log.info("方法参数类型:",Arrays.toString(parameterTypes));
        //5. 获取参数值
        Object[] args = jp.getArgs();
        log.info("方法参数值:",Arrays.toString(args));
    }
}

# 通知的执行顺序

当有多个切面的通知方法都被执行时，它们的执行顺序是怎样的呢？

  • 默认按照 bean 的名称字母排序
  • 用 @Order(数字)加在切面类上来控制顺序，数字越小，优先级越高，没有加数字的默认为int的最大值
