# SpringMVC

# 一、SpringMVC概述

三层架构

  • 表现层：负责请求处理和数据展示
  • 业务层：负责业务处理
  • 数据层：负责数据操作

MVC：一种用于Web应用程序的开发模式

  • Model（模型）：数据模型，用于数据处理，包括(service和dao)
  • View（视图）：页面视图，用于展示数据，jsp，html
  • Controller（控制器）：处理用户发送的请求，调用model完成数据处理，并对view视图进行响应

# 二、SpringMVC快速入门

  1. 快速入门

    1. 创建模块，打包方式选择 jar，勾选 Spring Web 支持
    2. 编写控制器，注意位置要在扫描范围内
    3. 在static目录中创建success.html页面
    4. 运行引导类，打开浏览器，输入访问地址访问控制器

  2. 也可以通过 jar 包运行程序

    • SpringBoot程序默认打jar包，当然也可以打war包，官方推荐打jar包。SpringBoot程序jar包中内置了web服务器，可以直接运 行jar包并访问程序。

    • 命令：java -jar jar包的名称

  3. 修改web程序虚拟目录和端口号

# 三、SpringMVC数据响应

SpringMVC可以响应不同类型的数据

  • 响应普通字符串
  • 响应json数据
  • 响应静态资源

# 响应普通字符串

对于普通字符串，在没有加@ResponseBody或者RestController注解的情况下，会默认去找return的返回值对应的文件路径，再进行页面跳转。

因此我们可以在方法或者Controller类上使用@ResponseBody注解或者@RestController，表示该方法不进行页面跳转，而是将返回值通过响应体响应给客户端。

@RestController 是一个组合注解，等同于 @Controller + @ResponseBody

//1. 获取普通类型参数
@RestController
@RequestMapping("/user")	//	一级请求目录，可加可不加
public class UserController {
	// @RequestMapping("/hello")	//	可以接收任意类型的请求
	@PostMapping("/hello")	//	只能接收POST类型的请求
	public String hello() {
		// return "/success.html";	//	在不使用@ResponseBody注解的时候会进行页面跳转
		return "hello SpringMVC";
	}
}

# 响应静态资源

对于放在static目录的静态资源，能够被 SpringBoot自动找到，因此对于静态文件尽量放在static目录中

从spring的源代码中可以看出以下目录都是可以自动识别到的

对于不在上述目录中的文件如果想被spring识别到需要实现WebMvcConfigurer接口配置静态资源访问路径

在resources目录下有一个和static同级的html目录，其中有一个hello.html文件，让spring能识别到它

@Configuration
public class WebMvcConfig implements WebMvcConfigurer {
	@Override
	public void addResourceHandlers(ResourceHandlerRegistry registry) {
		registry.addResourceHandler("/html/**").addResourceLocations("classpath:/hello/");
	}
}

需要注意的是，请求的参数一定要与请求方法中的参数一致，如果不一致就会匹配不到，也可以使用@RequestParam注解，将请求参数赋值给控制器方法中的形参。

@PostMapping("req1")
public String req1(@RequestParam(value="name") String username) {
	System.out.println(username);
	return "req1";

在上面的代码中请求传到后端的参数为username，最后@RequestParam会将username的值赋值给name属性。

@RequestParam注解的属性

  • value / name
  • required
  • defaultValue

@RequestParam的name属性和value属性是一样的效果，互为别名，

@RequestParam的required属性默认是true，表示这个参数必须传入，也可以通过将其设置为false来达到参数框架可加可不加的效果，如果不加即为null。

@RequestParam的defaultValue属性用来设置默认值，当请求参数不存在的时候，赋值为默认值发送请求。

对于@RequestParam注解不建议过多的使用，尽量保持请求参数和控制器中方法的参数一致，这样就没必要使用这个注解了。

# 请求Date类型的数据

  • SpringMVC默认支持转换的日期格式是yyyy/MM/dd。

  • 对于其他格式的日期参数可以使用@DateTimeFormat注解指定格式。

  • 如果封装到POJO中，需要在POJO属性上加上该注解

    @Data
@NoArgsConstructor
@AllArgsConstructor
public class User {
    private String name;  //姓名

    @JsonInclude(JsonInclude.Include.NON_NULL)  //  age属性不为空的时候才转换
    private Integer age;  //年龄

    @JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss", timezone = "GMT+8") // 设置Date响应参数
    @DateTimeFormat(pattern = "yyyy-MM-dd HH:mm:ss")    //  设置Date请求参数
    private Date birthday;  //生日

# 获取请求头和Cookie

@RequestHeader获取请求头、@CookieValue获取指定Cookie的值

@GetMapping("req5")
public String req5(@RequestHeader(value = "User-Agent",required = false) String userAgent,
                   @CookieValue(value = "JSESSIONID",required = false) String JSESSIONID) {
	System.out.println(userAgent);
	System.out.println(JSESSIONID);
	return "req5";
}

# 文件上传

文件上传对我们的表单是由要求的，下面是我们平时用的普通表单，无法完成文件上传

<form action="www.baidu.com" method="post">
  用户名：<input type="text" name="username" id="username">
  密码：<input type="password" name="password" id="password">
</form>

关于form的enctype属性

当 method 属性值为 post 时，enctype 就是将表单的内容提交给服务器的 MIME 类型 。可能的取值有：

  • application/x-www-form-urlencoded：未指定属性时的默认值。
  • multipart/form-data：当表单包含 type=file 的 <input> 元素时使用此值。
  • text/plain：出现于 HTML5，用于调试。这个值可被 <button>、<input type="submit"> 或 <input type="image"> 元素上的 formenctype 属性覆盖。

所以对于需要提交文件的表单需要设置enctype属性为multipart/form-data

<form action="http://localhost:8080/fileUpload" method="post" enctype="multipart/form-data">
  用户名：<input type="text" name="username" id="username"> <br>
  密码：<input type="password" name="password" id="password"> <br>
  文件上传：<input type="file" name="file" id="file">
</form>
