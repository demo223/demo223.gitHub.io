# Spring基础注解的使用

# 一、 用于创建对象的注解

当Spring扫描到以下注解时，就会创建Bean对象保存到Spring容器中

  1. @Controller 把控制器类交给 Spring管理
  2. @Service 把业务层类交给 Spring管理
  3. @Repository 把数据访问层类交给Spring管理
  4. @Component 把普通类交给 Spring管理，这个类不属于三层架构中的类

@SpringBootApplication 所在的类作为启动类(引导类)

Spring默认是加载（引导类、启动类）所在包及其子包中所有带有以上注解的类，创建这些类的对象，保存到Spring容器中。

如果Bean不在启动类所在包及其子包中，那么就需要在引导类上使用@ComponentScan("包路径")，不推荐使用。

@SpringBootApplication  //  Springboot主程序的入口
@ComponentScan(basePackages = "com")    //  指定Spring扫描包(为com包)及其子包中的所有类的注解

# 二、用于配置类的注解

@Configuration 用于指定当前类是一个配置类。

@Bean 将返回的对象保存到Spring容器中

如果要管理的对象来自于第三方JAR包，因为无法修改源代码，这时是无法用@Component等注解来实现的。

解决方案：使用@Configuration注解 + @Bean注解

@Configuration //表示该类是一个配置类，有SpringBoot自动扫描加载
public class DataSourceConfig {
    @Bean //默认添加到spring容器中Bean的名称就是方法名(首字母小写)
    public DruidDataSource dataSource(){
        //1 创建连接池对象
        DruidDataSource dataSource=new DruidDataSource();
        //2 设置连接参数
        dataSource.setDriverClassName("com.mysql.jdbc.Driver");
        dataSource.setUrl("jdbc:mysql://localhost:3306/brand_demo?useSSL=false");
        dataSource.setUsername("root");
        dataSource.setPassword("root");
        //3 返回连接池对象，保存到Spring容器中
        return dataSource;
    }
}

# 三、指定作用范围的相关注解

@Scope 用于指定bean的作用范围。

在类上使用@Scope注解定义Bean的作用域，Spring支持五种作用域，后三种在web环境才生效。

属性：value 指定范围的取值。常用取值：singleton(单例)和prototype(多例)。

  1. singleton：容器内同名称的 bean 只有一个实例（默认）
  2. prototype：每次使用该 bean 时会创建新的实例
  3. request：在 web 环境中，每个请求范围内会创建新的实例(不常用)
  4. session：在 web 环境中，每个会话范围内会创建新的实例(不常用)
  5. application：在 web 环境中，每个应用范围内会创建新的实例(不常用)

@Scope 注解默认情况下为单例模式，因此加不加@Scope("singleton)都是一样的。

# 四、生命周期的相关注解

@PostConstruct 用于指定初始化方法，在bean被创建之后调用

@PreDestory 用于指定销毁方法，singleton范围的bean的销毁方法会在容器关闭前被调用

@Lazy 创建时会在该bean在被第一次使用时创建，并且只创建一次，第二次及以后获取使用就不再创建

关于@Lazy注解：

​ 默认情况下 singleton范围的 bean 是容器创建时就会创建，如果希望用到时才创建，可以使用 @Lazy 注解标注在类上来延迟创建。

# 五、依赖注入的相关注解

  1. 给对象类型的属性赋值

    @Autowired 自动按照类型注入。只要容器中有唯一的一个bean对象类型和要注入的变量类型匹配，就可以注入成功。可以作用在变量或者方法上。

    • 加在成员变量上，会根据成员变量的类型到容器中找类型匹配的bean进行注入(赋值)

      @Controller
public class UserController {
	//	成员变量上面进行注入
	@Autowired
	private UserService userService;
	
	public void foo(){
		userService.foo();
	}
}

    @Autowired给属性赋值时，如果同类型的对象有多个，会按照变量名匹配，也可以使用@Qualifier注解指定要Bean的名称。

    1. @Qualifier 在按照类型注入的基础之上再按照名称注入，它在给类成员注入时要和@Autowired配合使用，但是在给方法参数注入是可以单独使用。

      问题：使用@Autowired给对象类型的属性赋值，如果同类型的对象有多个，赋值是否有问题？

      自动注入规则：使用@Autowired给对象类型的属性赋值，如果同类型的对象有多个，就按照变量名和Bean的名称进行匹配，建议使用@Qualifier注解指定要匹配的Bean的名称。

      //	UserService的实现类UserServiceImplA
@Service("UserServiceImplA")
public class UserServiceImplA implements UserService {
	@Override
	public void foo() {
		System.out.println("UserServiceImplA");
	}
}

//	UserService的实现类UserServiceImplB
@Service("UserServiceImplB")
public class UserServiceImplB implements UserService {
	@Override
	public void foo() {
		System.out.println("UserServiceImplB");
	}
}

//	测试类
@SpringBootTest
class SpringTestApplicationTests {
	@Autowired
	@Qualifier("UserServiceImplB")	//	表示这里注入的是UserServiceImplB对象
	private UserService userService;

	@Test
	public void test1(){
		userService.foo();	//	UserServiceImplB
	}
}

    1. 给简单属性赋值（基本类型、Sring类型）

      @Value 只能针对一个属性注入值，如果属性多了就比较麻烦

      @ConfigurationProperties 可以通过指定前缀来注入(前提：需要提供这些匹配的属性的getter和setter方法)

      例如，获取 application.properties 配置文件中的信息

      druid.driverClassName=com.mysql.jdbc.Driver
druid.url=jdbc:mysql:///ssm?useSSL=false
druid.username=root
druid.password=root

      配置类中注入相关属性

      @Configuration
public class ConfigTest {
	@Value("${druid.driverClassName}")
	private String driverClassName;
	@Value("${druid.url}")
	private String url;
	@Value("${druid.username}")
	private String username;
	@Value("${druid.password}")
	private String password;

	public void getInfo() {
		System.out.println(driverClassName);
		System.out.println(url);
		System.out.println(username);
		System.out.println(password);
	}
}
