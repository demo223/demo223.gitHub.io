# ArrayList和LinkedList

# List集合

List集合特点:

  1. ArrayList、LinekdList ：有序，可重复，有索引。
  2. 有序：存储和取出的元素顺序一致
  3. 有索引：可以通过索引操作元素 可重复：存储的元素可以重复

# ArrayList

ArrayList 类是一个可以动态修改的数组，与普通数组的区别就是它是没有固定大小的限制，我们可以添加或删除元素。ArrayList是一个数组队列，提供了相关的添加、删除、修改、遍历等功能。

ArrayList集合底层原理:

​ ArrayList底层是基于数组实现的：根据索引定位元素快，增删需要做元素的移位操作。

# ArrayList排序

ArrayList<Integer> list = new ArrayList<>();
list.add(13);
list.add(62);
list.add(23);
list.add(43);
list.add(37);
// naturalOrder是按照自然顺序(从小到大)排序的
list.sort(Comparator.naturalOrder());
System.out.println(list);   //  [13, 23, 37, 43, 62]
// reverseOrder是按照反向顺序(从大到小)排序的
list.sort(Comparator.reverseOrder());
System.out.println(list);   //  [62, 43, 37, 23, 13]

# LinkedList

LinkedList 类采用链表结构保存对象，这种结构的优点是便于向集合中插入或者删除元素。需要频繁向集合中插入和删除元素时，使用 LinkedList 类比 ArrayList 类效果高，但是 LinkedList 类随机访问元素的速度则相对较慢。这里的随机访问是指检索集合中特定索引位置的元素。

# LinkList集合的操作方法

LinkList的增删改查

LinkList除了包含List 接口中的常用方法外还有一些其他方法,如下表所示

例如 有这么一个LinkedList集合: LinkedList<Integer> list = new LinkedList<>();

操作 方法
将指定元素添加到此集合的开头 list.addFirst(元素)
将指定元素添加到此集合的末尾 list.addLast(元素)
返回此集合的第一个元素 list.getFirst()
返回此集合的最后一个元素 list.getLast()
删除此集合中的第一个元素(返回结果为被删除元素) list.removeFirst()
删除此集合中的最后一个元素(返回结果为被删除元素) list.removeLast()
LinkedList<Integer> list = new LinkedList<>();
list.add(1);
list.add(2);
list.add(3);
// 将指定元素添加到此集合的开头
list.addFirst(0);
// 将指定元素添加到此集合的末尾
list.addLast(100000);
// 返回此集合的第一个元素
System.out.println(list.getFirst());  //  0
// 返回此集合的最后一个元素
System.out.println(list.getLast());  //  10000
// 删除此集合中的第一个元素
System.out.println(list.removeFirst()); // 0 返回删除的第一个元素
// 删除此集合中的最后一个元素
System.out.println(list.removeLast());  // 10000 返回删除的最后一个元素
// 查询当前集合
System.out.println(list);   //  [1, 2, 3]

# ArrayList 类和 LinkedList 类的区别

每个类型什么时候使用？

使用 ArrayList 存储和访问数据，使用 LinkedList 操作数据。

相同点:

​ ArrayList 与 LinkedList 都是List 接口的实现类，因此都实现了 List 的所有未实现的方法，只是实现的方式有所不同。

不同点:

​ ArrayList 是基于动态数组数据结构的实现，访问元素速度优于LinkedList。LinkedList 是基于链表数据结构的实现，占用的内存空间比较大，但在批量插入或删除数据时优于 ArrayList。

使用场景:

​ 不同的结构对应于不同的算法，有的考虑节省占用空间，有的考虑提高运行效率，对于程序员而言，它们就像是“熊掌”和“鱼肉”，不可兼得。高运行速度往往是以牺牲空间为代价的，而节省占用空间往往是以牺牲运行速度为代价的。

​ 对于快速访问对象的需求，使用 ArrayList 实现执行效率上会比较好。需要频繁向集合中插入和删除元素时，使用 LinkedList 类比 ArrayList 类效果高。
