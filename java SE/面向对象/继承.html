# 继承

# 什么是继承?

Java中提供一个关键字extends，用这个关键字，我们可以让一个类和另一个类建立起父子关系。

为什么要使用继承?

可以提高代码的复用性，减少代码冗余，增强类的功能扩展性。

继承格式:

子类 extends 父类
1

继承的子类的特点:

  • 子类 继承父类，子类可以得到父类的属性和行为，子类可以使用。

  • Java中子类更强大

# 继承的特点

  1. 子类可以继承父类的属性和行为，但是子类不能继承父类的构造器。
  2. Java是单继承模式：一个类只能继承一个直接父类。
  3. Java不支持多继承、但是支持多层继承。
  4. Java中所有的类都是Object类的子类。
public class Test {
	public static void main(String[] args) {
		Dog dog = new Dog();
		// 子类访问父类的成员变量
		System.out.println(dog.animalName); //  父类成员变量
		// 子类访问父类的私有成员变量
		// System.out.println(dog.privateName);  //  不可访问
		// 子类访问父类的静态成员变量
		System.out.println(Dog.staticName); //  父类静态成员变量
		// 子类访问父类的静态成员方法
		Dog.eat();  //  父类的静态方法
		// 子类访问父类的成员方法
		dog.cry();  //  父类的成员方法
	}
}

子类是否可以继承父类的构造器？

​ 不可以的，子类有自己的构造器，父类构造器用于初始化父类对象。

子类是否可以继承父类的私有成员？

​ 可以的，只是不能直接访问。

子类是否可以继承父类的静态成员？

​ 子类可以直接使用父类的静态成员（共享）

​ 但个人认为：子类不能继承父类的静态成员。（共享并非继承）

# super关键字

# super调用父类构造方法

由于子类不能继承父类的构造方法，因此，如果要调用父类的构造方法，可以使用 super 关键字。super 可以用来访问父类的构造方法、普通方法和属性。

super 关键字可以在子类的构造方法中显式地调用父类的构造方法，基本格式如下：

super(参数);
1

注意: 这里的参数是父类构造方法中的参数。super( ) 必须是在子类构造方法的方法体的第一行。

声明父类 Person 和子类 Student，在 Person 类中定义一个带有参数的构造方法，代码如下：

class Person {
	String name = "父类属性";

	public Person(String name) {
		this.name = name;
	}
}

class User extends Person {
	// 这里默认会调用super()，但是如果Person父类中没有无参的构造方法，就会报错
}

上面的代码会出现编译错误，它会提示 : "没有可用的默认构造函数"。

构造器类型：

  • 无参数构造器:（默认存在的）：初始化对象时，成员变量的数据均采用默认值。
  • 有参数构造器：在初始化对象的时候，同时可以接收参数为对象进行赋值。

构造器的注意事项：

  • 任何类定义出来，默认就自带了无参数构造器，写不写都有。
  • 一旦定义了有参数构造器，那么无参数构造器就没有了，如果还想用无参数构造器，此时就需要自己手写一个无参数构造器了。

代码解释:

​ 这里上面的代码中父类Person中定义了一个有参的构造器，那么无参构造器就没有了(需要手动添加无参构造器)，而子类User中没有定义有参的构造函数，所以User中第一行代码会默认调用super()，所以这里会默认调用User的无参构造方法，然而Person中没有定义无参的构造器，所以这里会报错。

解决办法:

  1. 可以给父类添加无参构造方法
  2. 或者给子类添加有参的构造方法super(父类参数)，是的程序不再调用子类的默认无参构造方法super()。
class Person {
	String name = "父类属性";

	public Person() {
	}

	public Person(String name) {
		this.name = name;
	}
}

class User extends Person {
	// 如果子类中没有使用有参的构造方法，子类在这里默认会调用super()，
	// 但是如果Person父类中没有无参的构造方法，就会报错
	public int age = 12;

	public User(String name， int age) {
		super(name);
		this.age = age;
	}
}

public class Test {
    public static void main(String[] args) {
        User user = new User("张三",25);
        System.out.println(user);   //  User{age=25, name='张三'}
    }
}

# super访问父类成员

当子类的成员变量或方法与父类同名时，可以使用 super关键字来访问。如果子类重写了父类的某一个方法，即子类和父类有相同的方法定义，但是有不同的方法体，此时，我们可以通过 super 来调用父类里面的这个方法。

# super调用成员属性

使用 super 访问父类中的成员与 this关键字的使用相似，只不过它引用的是子类的父类，语法格式如下：

super.父类属性
1
class User extends Person {
	public int age = 12;
	public String name = "子类属性";

	public void foo() {
		/* 在不使用super访问name时，如果子类中也有name时，返回的是子类的name
		如果子类中没有name属性,那么访问属性时返回的是父类的name属性 */
		// 就近原则
		System.out.println(name);   //  子类属性
		System.out.println(super.name); //  父类属性
	}
}

# super调用成员方法

class Person {
	public void speak() {
		System.out.println("父类的speak方法");
	}
}

class User extends Person {
	public void speak() {
		System.out.println("子类的speak方法");
	}

	public void getSpeak() {
		speak();    //  调用子类自己的speak方法
		super.speak();  //  调用父类的speak方法
	}
}

# super和this的区别

super this
super.父类属性名：调用父类中的属性
super.父类方法名：调用父类中的方法
super()：调用父类的无参构造方法
super(参数)：调用父类的有参构造方法		 this.属性名：表示当前对象的属性
this.方法名(参数)：表示调用当前对象的方法

关于 super 和 this 关键字的异同，可简单总结为以下几条:

  1. 子类和父类中变量或方法名称相同时，用 super 关键字来访问。可以理解为 super 是指向自己父类对象的一个指针。在子类中调用父类的构造方法。
  2. this 是自身的一个对象，代表对象本身，可以理解为 this 是指向对象本身的一个指针。在同一个类中调用其它方法。
  3. this() 和 super() 不能同时出现在一个构造方法里面，因为 this()会调用它自己的无参构造方法，而它的无参构造方法中肯定会有 super() 语句的存在，所以在同一个构造方法里面有相同的语句，就失去了语句的意义，编译器也不会通过(相当于2次调用super())。
  4. this( ) 和 super( ) 都指的是对象，所以，均不可以在 static 环境中使用，包括 static 变量、static 方法和 static 语句块。

# 方法重写

子类重写父类方法，重写需要满足的条件

  1. 父类方法和子类方法同名，形参列表必须与被重写方法的名称和参数列表一致。

  2. 子类方法的访问权限 >= 父类方法的访问权限

  3. 子类返回类型需要 <= 父类的返回类型

    注意：java5 及更早版本返回类型要一样，java7 及更高版本可以不同

  4. 子类抛出的异常要 <= 父类抛出的异常

class Fu {
	public void test() throws Exception {
		System.out.println("父类方法");
		try {
			int a = 10 / 0;
		} catch (Exception e) {
			throw new Exception(e);
		}
	}
}

class Zi extends Fu {
	@Override
	public void test() throws RuntimeException {
		System.out.println("子类方法");
		try {
			int a = 10 / 0;
		} catch (Exception e) {
			throw new RuntimeException();
		}
	}
}

关于重载和重写

重载：发生在同一类中，方法名相同，参数列表不同。 重写：发生在父子类中，方法名相同，参数列表相同。 两同：①方法名相同 ②参数列表相同 两小：①子类返回值类型 小于等于 父类返回值类型（返回值为基本数据类型时，必须相等） ②子类抛出的异常 小于等于 父类抛出的异常 一大：子类访问控制修饰符 大于等于 父类访问控制修饰符
