# 简介

局部配置(只对当前实体类对象起作用)

在实体类的主键字段上面添加@TableId(type = IdType.AUTO)注解，表示设置为主键自增。

@Data
public class User {

    @TableId(type = IdType.AUTO)
    private Long id;
    
	// ...
}

关于@TableId注解的type属性的取值有以下5个，默认为

  • IdType.NONE 没有设置主键类型；跟随全局；全局的主键策略如果没有设置，默认是雪花算法
  • IdType.ASSIGN_ID 如果不设置 type 值，默认则使用 IdType.ASSIGN_ID 策略（自 3.3.0 起）。该策略会使用雪花算法自动生成主键 ID，主键类型为 Long 或 String（分别对应 MySQL 的表字段为 BIGINT 和 VARCHAR）
  • IdType.ASSIGN_UUID 如果使用 IdType.ASSIGN_UUID 策略，则会自动生成不含中划线的 UUID 作为主键。主键类型为 String，对应 MySQL 的表字段为 VARCHAR(32)
  • IdType.AUTO 对于像 MySQL 这样的支持主键自动递增的数据库，我们可以使用 IdType.AUTO 策略。
  • IdType.Input 插入前自行设置主键值

# 查询方法改进

  • 我们使用QueryWrapper来构建查询条件，操作的字段名以字符串的形式写死了，造成了硬编码。

  • mybatisplus提供了另一个类LambdaQueryWrapper来构建查询条件

需求8：使用LambdaQueryWrapper 完成查询 姓"张"的用户 并且年龄大于18

@Test
public void select8() {
	// 查询条件
	User user = new User();
	user.setName("张");
	user.setAge(18);
	// 1.创建LambdaQueryWrapper表达式
	LambdaQueryWrapper<User> lambdaQueryWrapper = new LambdaQueryWrapper<>();
	// 2.设置查询条件
	lambdaQueryWrapper.likeRight(user.getName() != null, User::getName, user.getName()).gt(user.getAge() != null,
			User::getAge, user.getAge());
	// 3.条件查询
	List<User> userList = userMapper.selectList(lambdaQueryWrapper);
	userList.forEach(System.out::println);
}

# Wrapper介绍 ：

  • Wrapper ： 条件构造抽象类，最顶端父类
  • AbstractWrapper ： 用于查询条件封装，生成 sql 的 where 条件
  • QueryWrapper ： Entity 对象封装操作类，不是用lambda语法
  • UpdateWrapper ： Update 条件封装，用于Entity对象更新操作
  • AbstractLambdaWrapper ： Lambda 语法使用 Wrapper统一处理解析 lambda 获取 column。
  • LambdaQueryWrapper ：看名称也能明白就是用于Lambda语法使用的查询Wrapper
  • LambdaUpdateWrapper ： Lambda 更新封装Wrapper

例如，使用 LambdaUpdateWrapper 完成 修改 张三的年龄为30，性别为女

# @TableField注解

功能1：比如说数据库中字段为create_time,而实体类的属性为createTime，前提是在全局策略配置中将驼峰命名关闭。

@TableField(value = "create_time")
private Date createTime;
1
2

功能2：又比如在实体类中有一个属性为updateTime，但是在数据库中没有这个字段，

但是在执行插入操作时给实体类的updateTime属性赋值了，那么可以通过在实体类的

updateTime属性上添加@TableField注解

@TableField(exist = false)
private Date updateTime;
1
2

执行代码的时候如果实体类中有数据库中不存在的字段就会报错，这个时候必须加上@TableField(exist = false)，这样就不会报错了。

例如查询所有的时候底层代码会默认执行

select  name,age, ... from tb_user;
1

会将表中所有的字段列举出来，而不是使用select *

功能3：在做日常数据操作时，还是有很多重复的操作

  • 新增时需要填充创建日期，修改日期
  • 修改时需要填充修改日期

【第一步】给pojo类中给需要自动填充的属性加上@TableField(fill = FieldFill.XXX)注解

// @TableField(value = "create_time")   //  设置数据库与实体类之间的映射
@TableField(fill = FieldFill.INSERT)    //  设置在插入数据的时候创建日期
private Date createTime;
// @TableField(exist = false)  //  表示该属性不是数据库中的字段
@TableField(fill = FieldFill.INSERT_UPDATE) //  设置在插入和修改数据的时候创建日期
private Date updateTime;

【第二步】自定义数据处理器MyMetaObjectHandler实现MetaObjectHandler接口。

@Component
@Slf4j
public class MyMetaObjectHandler implements MetaObjectHandler {
	@Override
	public void insertFill(MetaObject metaObject) {
		log.info("添加数据:公共字段自动填充");
		// setFieldValByName底层还是调用的metaObject.setValue，只是加了一些判断前提条件
		// metaObject.setValue("createTime", LocalDate.now());
		setFieldValByName("createTime", LocalDateTime.now(),metaObject);
		setFieldValByName("updateTime", LocalDateTime.now(),metaObject);
		// 从当前线程中获取用户信息
		Employee employee = ThreadContextUtil.getEmployee();
		setFieldValByName("createUser",employee.getId(),metaObject);
		setFieldValByName("updateUser",employee.getId(),metaObject);
	}


	@Override
	public void updateFill(MetaObject metaObject) {
		log.info("修改数据:公共字段自动填充");
		setFieldValByName("updateTime",LocalDateTime.now(),metaObject);
		// 从当前线程中获取用户信息
		Employee employee = ThreadContextUtil.getEmployee();
		setFieldValByName("updateUser",employee.getId(),metaObject);
	}
}
