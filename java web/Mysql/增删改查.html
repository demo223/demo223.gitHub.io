# DML

DML英文全称是Data Manipulation Language(数据操作语言)，用来对数据库中表的数据记录进 行增、删、改操作。 添加数据（INSERT） 修改数据（UPDATE） 删除数据（DELETE）

# 添加数据

  1. 给指定字段添加数据(添加一条不完整的数据到表中)

    insert into 表名 (字段1,字段2...) values (值1,值2,...);
    1

    这里使用value和values是一样的效果，別名罢了

  2. 给全部字段添加数据(添加一条完整的数据到表中)

    insert into 表名 values (值1,值2,值3)
    1

  3. 批量添加数据(添加多条数据)

    • 批量添加不完整的指定字段数据

      insert into demo(字段1,字段2) values (值1,值2),(值1,值2),(... , ...);
      1

      例如，向demo表中添加包含字段name和字段age的3条数据

      insert into demo(name,age)
values ('唐僧',11),
       ('猪八戒',22),
       ('沙和尚',33);
      1
      2
      3
      4

    • 批量添加完整的所有字段数据

      insert into 表名 values (字段1,字段2,字段3),(字段1,字段2,字段3),(... , ..., ...);
      1

      例如，向demo表中添加包含字段name，字段id和字段age的3条数据

      insert into demo
values ('刘备', 100, 44),
       ('关羽', 101, 55),
       ('张飞', 102, 66);
      1
      2
      3
      4

    注意事项：

    1. 插入数据时，指定的字段顺序需要与值的顺序是一一对应的。
    2. 字符串和日期型数据应该包含在引号中。
    3. 插入的数据大小，应该在字段的规定范围内。
    4. 对于not null的字段一定要插入数据。

# 修改数据

  1. 修改符合条件的数据(一般根据主键进行修改)

    update 表名 set 字段1 = 值1, 字段2 = 值2, ... where 条件;
    1

    例如，将demo表中id为101的数据的字段name和age进行修改为指定值

    update demo
set name = '关云长',
    age  = '45'
where id = 101;
    1
    2
    3
    4

  2. 修改表中所有的数据(不建议使用，软件会发出警告)

    update 表名 set 字段1 = 值1, 字段2 = 值2, ... ;
    1

    例如，将demo表中所有的age的值修改为18

    update demo set age = 18;
    1

    例如，将demo表中所有的age的值都加上10

    update demo set age = age + 10;
    1

注意事项：

修改语句的条件可以有，也可以没有，如果没有条件，则会修改整张表的所有数据。

# 删除数据

  1. 删除符合条件的数据

    delete from 表名 where 条件;
    1

    例如删除demo表中id值为6的数据

    delete
from demo
where id = 6;
    1
    2
    3

  2. 删除表中的所有数据(没有判断条件等于删除整张表)

    delete from 表名;
    1

注意事项：

  1. delete语句的条件可以有，也可以没有，如果没有条件，则会删除整张表的所有数据。
  2. delete语句不能删除某一个字段的值(可以使用update，将该字段值置为null即可)。
  3. 当进行删除全部数据操作时，datagrip会提示我们，询问是否确认删除。

# DQL

DQL英文全称是Data Query Language(数据查询语言)，数据查询语言，用来查询数据库中表的记录。

# 条件查询

条件查询的关键字是select

  1. 语法

    select 字段列表 from 表名 where 条件列表;
    1

  2. 条件

    常见的条件运算符如下

    比较运算符 功能
    > 大于
    < 小于
    <= 小于等于
    >= 大于等于
    = 等于
    <> 或 != 不等于
    between a and b 在指定范围内[a, b]
    in (...) 在列表中的指定值
    like 占位符(_匹配单个字符，%匹配多个字符) 模糊匹配
    is null 字段值是null
    is not null 字段值不是null

    常见的的逻辑运算符如下

    逻辑运算符 功能
    and或 && 并且 (多个条件同时成立)
    or 或 || 或者 (多个条件任意一个成立)
    not 或 ! 非，不是

# 分组查询

将查询结果按照1个或多个字段进行分组，字段值相同的为一组，其实就是按照某一列进行分类

语法

select 字段列表 表名 [where 条件] group by 分组字段名 [having 分组后过滤条件];
1

where与having区别：

  • 执行时机不同：where是分组之前进行过滤，不满足where条件的，不参与分组；而having是分组 之后对结果进行过滤。

  • 判断条件不同：where不能对聚合函数进行判断，而having可以。

注意：

  • 分组之后，查询的字段一般为聚合函数和分组字段，查询其他字段无任何意义。
  • 执行顺序: where > 聚合函数 > having 。
  • 支持多字段分组, 具体语法为 : group by 字段1, 字段2

查询表demo中年龄<=30，并根据性别分组，获取性别为"男"的性别的数量

select gender, count(*)
from demo
where age <= 30
group by gender
having gender = '男';

# 排序查询

排序在日常开发中是非常常见的一个操作，有升序排序，也有降序排序。

语法

select 字段列表 from 表名 order by 字段1 排序方式, 字段2 排序方式;
1

排序方式

  • ASC: 升序(默认值)
  • DESC: 降序

注意：

  • 如果是升序，可以不指定排序方式ASC
  • 如果是多字段排序，当第一个字段值相同时，才会根据第二个字段进行排序

将表demo中的数据按照工资进行排序，如果工资相同再按照性别降序排序

select name, id, age, salary, gender
from demo
order by salary, age desc;

# 分页查询

分页操作在业务系统开发时，也是非常常见的一个功能，我们在网站中看到的各种各样的分页条，后台 都需要借助于数据库的分页操作。

语法

select 字段列表 from 表名 limit 起始索引, 查询记录数;
1

注意：

  • 起始索引从0开始，起始索引 = （查询页码 - 1） 每页显示记录数*。
  • 分页查询是数据库的方言，不同的数据库有不同的实现，MySQL中是limit。
  • 如果查询的是第一页数据，起始索引可以省略，直接简写为 limit 10。
  • 第一页是limit 0,10 第二页就是limit 10,10 第三页就是limit 20,10

将表demo中的数据按照每页10条进行排序并返回第二页的数据

select name, id, age, salary, gender
from demo
limit 10,10;

# DQL语句的执行顺序

DQL语句在执行时的执行顺序，也就是先执行那一部分，后执行那一部分。

DQL的语句执行顺序(从左至右)为

表名 条件语句 分组 分组过滤 字段列表 排序 分页
from where group by having select order by limit

# DCL

DCL英文全称是Data Control Language(数据控制语言)，用来管理数据库用户、控制数据库的访 问权限。

# 管理用户

在MySQL中需要通过用户名@主机名的方式，来唯一标识一个用户

主机名也可以使用 % 通配

  1. 查询用户

    select * from mysql.user;
    1

  2. 创建用户

    create user '用户名'@'主机名' identified by '密码';
    1

    创建一个用户名为test，主机名为localhost的用户

    create user 'test'@'localhost' identified by '123456';
    1

  3. 修改用户密码

    set password for 用户名@主机名 = password ('修改后的密码');
    1

    修改用户名为test，主机名为localhost的用户的密码为666666

    set password for test@localhost= password ('666666');
    1

  4. 删除用户

    drop user '用户名'@'主机名'
    1

    删除用户名为test，主机名为localhost的用户

    drop user 'test'@'localhost';
    1

注意：

  • 主机名为localhost的用户，只能在当前主机访问数据库
  • 主机名为%的用户，可以在所有主机访问数据库

# 权限控制

MySQL的常用权限

权限 说明
all，all privileges 所有权限
select 查询数据
insert 插入数据
delete 删除数据
alter 修改表
drop 删除数据库/表/视图
create 创建数据库/表

  1. 查询权限

    show grants for 'root'@'localhost';
    1

    例如查看root'@'localhost用户的权限

    show grants for 'root'@'localhost';
    1

    查询结果如下：表示该用户拥有所有权限

    GRANT ALL PRIVILEGES ON `study`.`demo` TO 'test'@'localhost'
    1

  2. 授予权限

    grant 权限列表 on 数据库名.表名 to '用户名'@'主机名';
    1

    例如

    1. 给用户test'@'localhost授予访问数据库study中表demo的权限

      grant all on study.demo to 'test'@'localhost';
      1

    2. 给用户test'@'localhost授予访问数据库study中所有表的权限

      grant all on study.* to 'test'@'localhost';
      1

  3. 撤销用户的权限

    revoke 权限列表 on 数据库名.表名 from '用户名'@'主机名';
    1

    例如撤销用户test'@'localhost的all权限

    revoke all on study.demo from 'test'@'localhost';
    1
