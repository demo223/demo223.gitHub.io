# 新的日期时间类

# 介绍

LocalDate、LocalTime 和 LocalDateTime是 Java.time API 中的类，分别表示没有时区信息的日期、时间和日期/时间。这些类属于 java.time 包，它是在 Java 8 中引入的，取代了旧的 java.util.Date 和 java.util.Calendar 类。

下面是这些类的简要介绍：

  • LocalDate 表示没有时区信息的日期（年、月、日）。您可以使用此类表示日期，例如一个人的生日或一个活动。
  • LocalTime 表示没有时区信息的时间（小时、分钟、秒、纳秒）。您可以使用此类表示时间，例如会议时间或航班出发时间。
  • LocalDateTime 表示没有时区信息的日期和时间（年、月、日、小时、分钟、秒、纳秒）。您可以使用此类表示特定的时间点，例如银行交易或日志条目。
public class Test {
	public static void main(String[] args) {
		System.out.println(LocalDate.now());    //  2022-12-31
		System.out.println(LocalTime.now());    //  22:59:26.985695500
		System.out.println(LocalDateTime.now());    //  2022-12-31T22:59:26.985695500
	}
}

# 时间的修改(计算)

# 时间计算

LocalDate、LocalTime、LocalDateTime、Instant为不可变对象，修改这些对象对象会返回一个副本

例如LocalDate的计算，你可以使用plusDays()、plusWeeks()、plusMonths()、plusYears()等方法来在指定日期上加上时间量。

LocalDate date1 = LocalDate.of(2023, 01, 01);
1

例如，计算100天后的时间

LocalDate date2 = date1.plusDays(100);
System.out.println(date2);  //  2023-04-11
1
2

LocalDate类还有其他以with开头的方法，用于修改日期中的其他部分。例如：

  • withMonth(int month)：修改日期中的月份部分
  • withDayOfMonth(int dayOfMonth)：修改日期中的天数部分
  • withDayOfYear(int dayOfYear)：修改日期中的年中天数部分

LocalTime类也有类似的方法，用于修改时间中的各个部分。例如：

  • withHour(int hour)：修改时间中的小时部分
  • withMinute(int minute)：修改时间中的分钟部分
  • withSecond(int second)：修改时间中的秒数部分
  • withNano(int nanoOfSecond)：修改时间中的纳秒部分

像LocalDate和LocalTime等都是不可变的类。这意味着一旦创建了对象，就无法更改对象的状态。

因此，当你想要修改日期或时间时，你需要创建新的对象。

例如，计算指定日期是这一年的第几天，这个月的第几周，星期几等

LocalDate date = LocalDate.of(2023, 05, 18);    //  2023-05-18
int dayOfYear = date.getDayOfYear();    //  138	2023年的第138天
int dayOfMonth = date.getDayOfMonth();  //  18	05月的第18天
DayOfWeek dayOfWeek = date.getDayOfWeek();  //  THURSDAY	//	当天是星期四(THURSDAY)

DateTimeFormatter和SimpleDateFormat区别

DateTimeFormatter和SimpleDateFormat都是用来格式化日期和时间的类，但是它们有一些重要的区别。

  • DateTimeFormatter是线程安全的，可以在多线程环境中使用。而SimpleDateFormat是非线程安全的，如果在多线程环境中使用，可能会出现问题。在多线程环境中使用 SimpleDateFormat 时，你可以使用 synchronized 块来保证同一时刻只有一个线程在使用它。这样就可以避免线程安全问题了。

  • DateTimeFormatter提供了更多的格式化选项，如本地化格式、带毫秒的时间格式等。而SimpleDateFormat的格式化选项较少。

@JsonFormat和@DateTimeFormat区别

@JsonFormat 和 @DateTimeFormat 都是用来格式化日期的注解，但是它们的使用场景和作用略有不同。

  • @JsonFormat 注解是由 Jackson 框架提供的，用于格式化日期字段。它通常用于将 Java 对象序列化为 JSON 字符串时使用。

  • @DateTimeFormat 注解是由 Spring 框架提供的，用于格式化日期字符串。它通常用于将日期字符串转换为日期类型时使用。

  • @DatetimeFormat是将String转换成Date，一般前台给后台传值时用

    @JsonFormat 将Date转换成String， 一般后台传值给前台时

总的来说，@JsonFormat 注解用于将日期序列化为字符串，而 @DateTimeFormat 注解用于将日期字符串反序列化为日期对象。

# 关于前后端的对象的(反)序列化

序列化

如果你想在前端传递一个对象到后端，通常需要将对象序列化为字符串后再传递。

前端	->	对象	->	序列化	->	字符串	->	后端
1

如果你想将后端的 Java 对象传递给前端，通常需要将对象序列化为字符串后再传递。

后端	->	对象	->	序列化	->	字符串	->	前端
1

也就是说，前后端对象的传输通常都是通过 JSON 字符串来实现的

反序列化

我们这里就只考虑后端的反序列化，后端Spring中默认使用Jackson将 JSON 字符串反序列化为 Java 对象

前端	->	字符串	->	后端	->	反序列化	->	对象
1

Spring中一般我们都是通过注解实现序列化和反序列化

例如 @RequestBody 注解用于反序列化请求体，而 @RestController 注解用于将方法返回的对象序列化为响应体。

序列化中存在的问题

有个地方需要注意的是，Jackson会自动将字段名的首字母转化为小写，例如 XAxis 我们表示 X轴字段

但是在序列化或变成了 xaxis
