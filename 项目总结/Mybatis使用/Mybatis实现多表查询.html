# Mybatis实现多表查询

Mybatis 是一种 Java 持久化框架，可以很方便地实现多表查询。下面介绍两种常见的方法。

有两个表 user 和 order，其中 user 表的主键是 id，order 表的外键是 user_id

user表

id name email
1 张三 zhangsan@example.com
2 李四 lisi@example.com
3 王五 wangwu@example.com

order表

id product_name price user_id
1 商品 A 100.0 1
2 商品 B 200.0 1
3 商品 C 300.0 2
4 商品 D 400.0 3
5 商品 E 500.0 3

# 嵌套查询

想要查询某个用户以及他所有的订单，可以创建一个嵌套查询。具体步骤如下：

  • 创建一个包含用户信息的 POJO 类 User，和一个包含订单信息的 POJO 类 Order。
@Data
@NoArgsConstructor
@AllArgsConstructor
public class Order {
    private Integer id;
    private String productName;
    private Double price;
}

  • 创建一个 SQL 查询，该查询包含一个嵌套查询来检索指定用户的所有订单。例如：
SELECT u.id,
       u.NAME,
       u.email,
       o.id AS order_id,
       o.product_name,
       o.price
FROM USER u
         LEFT JOIN `order` o ON u.id = o.user_id
WHERE u.id = #{userId}

  • 在 Mybatis 的 Mapper XML 文件中定义该 SQL 查询，并使用 <resultMap> 元素将结果映射到 Java 对象中。例如：
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN" "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="com.zq.mapper.OrderMapper">

    <resultMap id="userOrderMap" type="com.zq.entity.User">
        <id property="id" column="id"/>
        <result property="name" column="name"/>
        <result property="email" column="email"/>
        <collection property="orders" ofType="com.zq.entity.Order">
            <id property="id" column="order_id"/>
            <result property="productName" column="product_name"/>
            <result property="price" column="price"/>
        </collection>
    </resultMap>

    <select id="getUserWithOrders" resultMap="userOrderMap">
        SELECT u.id,
               u.NAME,
               u.email,
               o.id AS order_id,
               o.product_name,
               o.price
        FROM USER u
                 LEFT JOIN `order` o ON u.id = o.user_id
        WHERE u.id = #{userId}
    </select>
</mapper>

# 带有关联查询的嵌套结果

这种方法使用 Mybatis 的 association 和 collection 元素来进行多表查询。假设有两个表 user 和 order，并且你想要查询所有订单和它们对应的用户信息，可以创建一个带有关联查询的嵌套结果。具体步骤如下：

  • 创建一个包含用户信息的 POJO 类 User，和一个包含订单信息的 POJO 类 Order。在 Order 类中增加一个 User 类型的属性，用于存储与该订单相关联的用户信息。
@Data
@NoArgsConstructor
@AllArgsConstructor
public class Order {
    private Integer id;
    private String productName;
    private Double price;
    private User user;	//	与上面不同的是这里添加了一个user对象，订单对应的用户信息
}

  • 创建一个 SQL 查询，该查询包含一个关联查询来检索所有订单及其对应的用户信息。例如：

  •   SELECT o.id as order_id, o.product_name, o.price, 
         u.id as user_id, u.name, u.email 
  FROM `order` o 
  INNER JOIN user u ON o.user_id = u.id

    SQL查询结果如下

    id product_name price user_id name email
    1 商品 A 100.0 1 张三 zhangsan@example.com
    2 商品 B 200.0 1 张三 zhangsan@example.com
    3 商品 C 300.0 2 李四 lisi@example.com
    4 商品 D 400.0 3 王五 wangwu@example.com
    5 商品 E 500.0 3 王五 wangwu@example.com

  • 在 Mybatis 的 Mapper XML 文件中定义该 SQL 查询，并使用 <resultMap> 元素将结果映射到 Java 对象中。在 Order 类的 <resultMap> 元素中，使用 association 元素表示与订单相关联的用户信息；在 User 类的 <resultMap> 元素中，使用 collection 元素表示与用户相关联的所有订单信息。例如：

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN" "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="com.zq.mapper.OrderDao">

    <resultMap id="orderWithUserMap" type="com.zq.entity.Order">
        <id property="id" column="order_id"/>
        <result property="productName" column="product_name"/>
        <result property="price" column="price"/>
        <association property="user" javaType="com.zq.entity.User">
            <id property="id" column="user_id"/>
            <result property="name" column="name"/>
            <result property="email" column="email"/>
            <collection property="orders" ofType="com.zq.entity.Order">
                <id property="id" column="order_id"/>
                <result property="productName" column="product_name"/>
                <result property="price" column="price"/>
            </collection>
        </association>
    </resultMap>

    <select id="getOrderWithUser" resultMap="orderWithUserMap">
        SELECT o.id as order_id,
               o.product_name,
               o.price,
               u.id as user_id,
               u.name,
               u.email
        FROM `order` o
                 INNER JOIN user u ON o.user_id = u.id
    </select>
</mapper>

# 总结

在项目中，某些实体类之间肯定有关联关系，比如一对一，一对多等。

mybatis中使用 association和 collection。 association：一对一关联 collection：一对多关联

collection：一对多关联

id name email order_id product_name price
3 王五 wangwu@example.com 4 商品 D 400.0
3 王五 wangwu@example.com 5 商品 E 500.0

association：一对一关联

id product_name price user_id name email
1 商品 A 100.0 1 张三 zhangsan@example.com
