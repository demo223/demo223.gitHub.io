# SQL33 找出每个学校GPA最低的同学

题目：现在运营想要找到每个学校gpa最低的同学来做调研，请你取出每个学校的最低gpa。

示例：user_profile

id device_id gender age university gpa
1 2138 male 21 北京大学 3.4
2 3214 male 复旦大学 4
3 6543 female 20 北京大学 3.2
4 2315 female 23 浙江大学 3.6
5 5432 male 25 山东大学 3.8
6 2131 male 28 山东大学 3.3
7 4321 female 26 复旦大学 3.6

根据示例，你的查询结果应参考以下格式，输出结果按university升序排序：

device_id university gpa
6543 北京大学 3.2000
4321 复旦大学 3.6000
2131 山东大学 3.3000
2315 浙江大学 3.6000

我的解答：

select
    device_id,
    university,
    min(gpa) as gpa
from
    user_profile
group by
	device_id,
    university
order by
    university

select
    device_id,
    university,
    gpa
from
    user_profile as t1
where
    gpa = (
        select
            min(gpa)
        from
            user_profile as t2
        where
            t1.university = t2.university

order by
    university

如果只执行了这个 SQL 查询语句，那么查询结果就只会返回一个值，即 user_profile 表中 GPA 的最低值。不过，我们在原始 SQL 查询语句中使用了该子查询语句作为 SELECT子句中的一部分，以便计算每个大学对应的最低 GPA值。

当查询语句执行时，主查询会对 user_profile 表中的每一行进行扫描，并在子查询（即 SELECT子句中嵌套的子查询）中找到与该行所对应学校相同的所有行，并从中找到 GPA最低的行。因此，主查询中的结果集将包含每个大学对应的最低 GPA值，而非整个表的最低 GPA值。

这个 SQL 查询中，我们使用了两个表别名：t1 和 t2。在主查询中，我们选取 user_profile 表中所有符合条件的行，即 GPA值等于该学校中最低 GPA的行。为了找到每个学校对应的最低 GPA值，我们使用了一个子查询（即 SELECT子句中嵌套的子查询），该子查询会返回当前行所对应学校的最低 GPA值。

子查询中使用了与主查询中相同的表别名 t2，并使用 WHERE子句筛选出与当前行所对应学校相同的所有行，然后使用 MIN函数找到 GPA最低的那一行。由于子查询使用了主查询中的某些值（即使用 t1.university 与 t2.university 进行关联），因此该子查询被称为“关联查询”。

当主查询执行时，对于 user_profile 中的每一行，它都会执行子查询以获取与该行所对应学校的最低 GPA值相同的所有行。如果找到多个行，则主查询将选取其中任意一行作为结果；如果未能找到匹配行，则该行将不会包含在结果集中。

# SQL34 统计复旦用户8月练题情况

题目： 现在运营想要了解复旦大学的每个用户在8月份练习的总题目数和回答正确的题目数情况，请取出相应明细数据，对于在8月份没有练习过的用户，答题数结果返回0.

示例：用户信息表user_profile

id device_id gender age university gpa active_days_within_30
1 2138 male 21 北京大学 3.4 7
2 3214 male 复旦大学 4.0 15
3 6543 female 20 北京大学 3.2 12
4 2315 female 23 浙江大学 3.6 5
5 5432 male 25 山东大学 3.8 20
6 2131 male 28 山东大学 3.3 15
7 4321 female 26 复旦大学 3.6 9

示例：question_practice_detail

id device_id question_id result date
1 2138 111 wrong 2021-05-03
2 3214 112 wrong 2021-05-09
3 3214 113 wrong 2021-06-15
4 6543 111 right 2021-08-13
5 2315 115 right 2021-08-13
6 2315 116 right 2021-08-14
7 2315 117 wrong 2021-08-15
……

根据示例，你的查询应返回以下结果：

device_id university question_cnt right_question_cnt
3214 复旦大学 3 0
4321 复旦大学 0 0

我的SQL如下

select
    u.device_id,
    u.university,
    count(q.device_id) as question_cnt,
    sum(if (result = 'right', 1, 0)) as right_question_cnt
from
    user_profile as u
    left join question_practice_detail as q on u.device_id = q.device_id
where
    u.university = '复旦大学'
    and month (q.date) = 8
group by
    u.device_id

这两个 SQL 查询的区别在于左连接 question_practice_detail 表和主查询的时机不同。

第一个 SQL 查询中，左连接发生在 WHERE子句之前，使用了一个 ON子句来过滤月份为 8 的记录，如下所示：

left join question_practice_detail as q on u.device_id = q.device_id
    and month (q.date) = 8
1
2

而第二个 SQL 查询中，则是在 WHERE子句中使用了与 ON子句相同的过滤条件，如下所示：

where u.university = '复旦大学' and month(q.date) = 8
1

从功能上来说，这两个查询是等效的。它们都返回了在 user_profile 表中，大学为“复旦大学”且在 8 月份有做过练习题的用户信息，包括设备号、大学名称、答题次数以及回答正确的次数。在 GROUP BY 子句中，用户信息按设备号进行分组，因此每个设备只会出现一次。

需要注意的是，在第一个查询中，ON子句中的筛选条件（即 month(q.date) = 8）作用于左连接的结果集，而不是主查询的结果集。因此，如果某个用户在该月没有做过练习题，则该用户的信息仍然会出现在结果集中，但是 question_cnt 和 right_question_cnt 会被设置为 NULL。在第二个查询中，由于 WHERE子句中的条件过滤了未做练习题的用户，因此不会出现这种情况。

左连接和 WHERE连接查询的区别在于左连接保留了左表中所有的记录，而WHERE 连接只返回符合条件的记录。

左连接是指在连接两个表时，保留左表中的所有记录，不管右表中是否有匹配的记录。这意味着，在左表中没有匹配到右表中的记录时，对应的结果集中会使用 NULL 值来填充右表中相关的列的值。左连接通常用于需要包含左表中所有数据的查询，以及查询两个表之间关系的查询。

WHERE连接是指在连接两个表时，使用 WHERE子句来过滤数据并仅返回符合条件的记录。这种连接只返回两个表之间匹配的记录，因此结果集将小于两个表中任何一个表的总记录数。WHERE 连接通常用于需要从多个表中选择特定条件的查询。

例如，假设我们有两个表 A和 B。如果我们要找出 A中所有记录，无论它是否存在于 B中，我们可以使用左连接。左连接将从 A 中选择所有记录，并为 B中所有匹配的记录填入相应的列。如果我们只需要选择 A 和 B中都存在的记录，则可以使用 WHERE连接，在 WHERE子句中指定匹配条件。

总之，左连接和 WHERE连接的使用取决于需求。**左连接适用于需要包含左表中所有数据的查询，而WHERE连接适用于需要选择特定条件的查询。**如果您需要从两个表中进行选择，则可以使用其中任何一种连接类型。
