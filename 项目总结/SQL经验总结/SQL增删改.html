# 表格数据批量插入

在 MySQL 中将一个表的数据导入另一个相同表结构的表，可以使用 INSERT INTO SELECT 语句或者 INSERT INTO VALUES 语句。具体操作步骤如下：

  • INSERT INTO SELECT
  • INSERT INTO VALUES

  1. 使用 INSERT INTO SELECT 语句导入数据，例如：

    INSERT INTO new_table SELECT * FROM old_table;
    1

    这个语句将从 old_table 表中选取所有列和行并插入到 new_table 中，因为新表和旧表结构相同，所以数据会被正确地复制。

    如果需要选择部分列数据进行复制，并且列名在新旧表中相同，可以使用以下语句：

    INSERT INTO new_table (col1, col2, col3) SELECT col1, col2, col3 FROM old_table;
    1

    这个语句将从 old_table 表中选取 col1、col2 和 col3 列数据并插入到 new_table 中。

  2. 使用 INSERT INTO VALUES 语句导入数据，例如：

    INSERT INTO new_table (col1, col2, col3) VALUES ('val1', 'val2', 'val3'), ('val4', 'val5', 'val6');
    1

    这个语句将直接将指定的值插入到 new_table 表中。需要注意的是，如果有自增列，则插入时需要显式指定值或像上述 INSERT INTO SELECT 语句一样省略该列。

总结：INSERT INTO VALUES 适合于直接插入少量数据，而 INSERT INTO SELECT 更适合于从一个表中选择大量数据并将它们插入到另一个表中。

replace into 跟 insert into功能类似，不同点在于：replace into 首先尝试插入数据到表中，如果发现表中已经有此行数据（根据主键或者唯一索引判断）则先删除此行数据，然后插入新的数据；否则，直接插入新数据。

# SQL更新

现有一张试卷信息表examination_info，表结构如下图所示：

Filed Type Null Key Extra Default Comment
id int(11) NO PRI auto_increment (NULL) 自增ID
exam_id int(11) NO UNI (NULL) 试卷ID
tag char(32) YES (NULL) 类别标签
difficulty char(8) YES (NULL) 难度
duration int(11) NO (NULL) 时长
release_time datetime YES (NULL) 发布时间

请把examination_info表中tag为PYTHON的tag字段全部修改为Python。

update
    examination_info
set
    tag = 'Python'
where
    tag = 'PYTHON'

也可以使用replace

update examination_info 
set tag = replace(tag,'PYTHON','Python')

replace与update的区别：

  1. update在没有匹配记录时什么都不做，而replace在有重复记录时更新，在没有重复记录时插入。
  2. update可以选择性地更新记录的一部分字段， 而replace在发现有重复记录时就将这条记录彻底删除，再插入新的记录。
