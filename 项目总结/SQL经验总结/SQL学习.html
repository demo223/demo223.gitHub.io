这两个 SQL 查询语句的作用是相同的，都是从用户配置表 user_profile中获取设备 ID(device_id)、年龄(age)和就读大学(university)信息，其中大学名称包含"北京"字样。

两者的区别在于匹配方式不同。第一个查询语句使用了正则表达式(REGEXP)，第二个查询语句使用了通配符(%)。

具体来说，正则表达式(REGEXP)是一种强大的文本匹配工具，它可以对字符串进行高度灵活的匹配操作，支持各种规则和模式的匹配。在上述查询语句中，采用 REGEXP匹配方式，查询结果会把大学名称恰好为"北京"的记录返回，也包括大学名称中间包含"北京"的记录。

而通配符(%)则是 SQL 中常用的通配符。在上述查询语句中，使用了 like '%北京%' 的方式进行匹配，这意味着只要大学名称包含"北京"这个字符串，就会被查询出来。例如，不仅能够匹配"北京大学"，还能匹配"中国传媒大学北京分校"这样的字段。

综上所述，二者的区别在于匹配方式不同，针对不同的文本匹配需求，我们可以灵活选择合适的匹配方式来完成查询操作。需要注意，在数据量较大的情况下，REGEXP 在性能上可能会受到影响，如有需要，可以考虑使用全文搜索的方式来进行匹配。

# having使用

SELECT
    university,
    ROUND(AVG(question_cnt), 3) AS avg_question_cnt,
    ROUND(AVG(answer_cnt), 3) AS avg_answer_cnt
FROM
    user_profile
GROUP BY
    university
HAVING
    AVG(question_cnt) < 5
    OR AVG(answer_cnt) < 20;

HAVING子句可以在 GROUP BY 子句之后进行使用，用于过滤分组后的数据。在上述 SQL 查询语句中，通过子查询得到每个大学(question_cnt和answer_cnt)的平均问题数和平均回答数，并将结果作为临时表 t 进行过滤，保留符合条件的结果。

与原始 SQL 查询语句相比，新的查询语句使用了 HAVING子句来进行过滤，使得查询语句更加简洁。需要注意的是，HAVING子句只能用于聚合函数，所以在本例中，我们可以将 WHERE子句中的条件改写为 AVG(question_cnt) 和 AVG(answer_cnt)。

  1. HAVING用于对分组后的结果再进行过滤，它的功能有点像WHERE，但它用于组而不是单个记录。

  2. 在HAVING中可以使用统计函数，但在WHERE中则不能。

  3. HAVING通常与GROUP BY一起使用。

# if使用

IF函数用于对条件进行判断并返回相应的值。其语法如下：

IF(expr1, expr2, expr3)
1

其中，expr1 是要求值的条件，expr2 是在条件求值为 TRUE时返回的值，expr3 是在条件求值为 FALSE 时返回的值。

用处：当从数据库中查询出来的结果需要转换成中文或是其他自定义的格式显示在网页上的时候，可以直接在SQL中处理。

select
    if (il.status_id = 'INV_STTS_AVAILABLE', '全新', '二手') as status_id
from
    inventory_location as il;

在 SQL 查询语句中，SELECT子句用于指定要查询的列或表达式。其基本语法如下：

SELECT column1, column2, ..., columnN
FROM table_name;
1
2

其中，column1, column2, ..., columnN 是要查询的列名或表达式，可以使用任何合法的表达式来代表一个列名。这些表达式可以是算术运算，逻辑运算，字符串操作，函数调用等，甚至包括常量和子查询。因此，在 SELECT子句中并不仅限于数据库表中已存在的字段名，而可以使用任何合法的表达式来查询数据。

举例来说，如果你想查询一个数字列的平方值和立方值，可以使用如下 SELECT语句：

SELECT col1, col1 * col1 as square, col1 * col1 * col1 as cube
FROM table_name;
1
2

在此查询中，col1 * col1 和 col1 * col1 * col1 分别代表该列的平方值和立方值，这两个表达式并不是数据库表中已经存在的字段名，但它们是合法的表达式，因此可以在 SELECT子句中进行使用。

需要注意的是，如果表达式在 SELECT子句中包含了多个列或涉及到了多个表，则应当使用表别名来限定该表达式中各个列所属的表，以避免歧义。

# case when使用

在 MySQL 中，CASE WHEN 是一种条件表达式，用于在查询结果中根据条件进行值的转换。其基本语法如下：

CASE WHEN condition1 THEN result1
     WHEN condition2 THEN result2
     ...
     ELSE resultN
END
1
2
3
4
5

其中，condition1, condition2, ... 是要比较的条件，通常使用比较运算符或逻辑运算符进行比较，例如 age < 18 或者 name = '张三'。result1, result2, ..., resultN 是与每个条件关联的结果，可以是任何数据类型，包括常量、列名、表达式等。最后，ELSE 子句是可选的，用于在所有条件不满足时返回的默认结果。

举例来说，如果你想查询学生的分数，同时将他们的成绩分为优秀、良好、及格和不及格四个等级，可以使用如下 SQL 语句：

SELECT name, score,
    CASE
        WHEN score >= 90 THEN '优秀'
        WHEN score >= 80 THEN '良好'
        WHEN score >= 60 THEN '及格'
        ELSE '不及格'
    END as grade
FROM student;

# timestampdiff使用

TIMESTAMPDIFF() 是 MySQL 中用于计算两个日期或时间之间的差异的函数。它可以返回两个日期之间的差值，以秒、分钟、小时、天数等形式表示。

以下是 TIMESTAMPDIFF() 函数的语法：

TIMESTAMPDIFF(unit, datetime1, datetime2)
1

其中，unit 参数指定要返回的时间单位，可以是以下字符串中的一种：

  • MICROSECOND（微秒）
  • SECOND（秒）
  • MINUTE（分钟）
  • HOUR（小时）
  • DAY（天）
  • WEEK（周）
  • MONTH（月）
  • QUARTER（季度）
  • YEAR（年）

datetime1 和 datetime2 参数是要比较的日期/时间或表达式。它们必须有相同的数据类型。

以下是一些示例用法：

-- 计算两个日期之间的天数差
SELECT TIMESTAMPDIFF(DAY, '2023-05-01', '2023-05-31'); -- 输出 30

-- 计算两个时间之间的小时差
SELECT TIMESTAMPDIFF(HOUR, '2023-05-29 09:00:00', '2023-05-29 12:00:00'); -- 输出 3

-- 计算两个时间之间的分钟差
SELECT TIMESTAMPDIFF(MINUTE, '2023-05-29 09:00:00', '2023-05-29 09:15:00'); -- 输出 15
