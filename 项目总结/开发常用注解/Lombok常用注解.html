# Lombok注解

# JavaBean中的注解

  1. @EqualsAndHashCode(callSuper = false)：这是Lombok框架中的注解，它会自动生成equals和hashCode方法。在这里，它的作用是生成SysResource类的equals和hashCode方法，并且==不包括父类的属性==。
  2. @Accessors(chain = true)：这是Lombok框架中的注解，它用于配置链式调用。通过在类上添加@Accessors注解，并将chain属性设置为true，可以启用链式调用。在这里，它的作用是启用SysResource类的链式调用。
  3. @TableName("sys_resource")：这是Mybatis-Plus框架中的注解，它用于指定实体类对应的数据库表名。在这里，它的作用是将SysResource类对应到数据库中的sys_resource表
@Data
@EqualsAndHashCode(callSuper = false)
@Accessors(chain = true)
@TableName("sys_resource")
public class Resource implements TreeModel<Resource>, Serializable {

    private static final long serialVersionUID = 1L;

    /**
     * 主键id
     */
    private String id;

    /**
     * 父级id
     */
    private String parentId;
    
	//	数据库字段省略......
    
    /**
     * 子级数据
     */
    @TableField(exist = false)
    private List<Resource> children;

}

@TableField(exist = false) 注解加载bean属性上，表示当前属性不是数据库的字段，但在项目中必须使用，这样在新增等使用bean的时候，mybatis-plus就会忽略这个，不会报错

# @EqualsAndHashCode注解

这是Lombok框架中的注解，它会自动生成equals和hashCode方法。callSuper属性用于指定是否包含父类的属性，默认值为false。当callSuper为false时，equals和hashCode；方法不会包括父类的属性当callSuper为true时，equals和hashCode方法会包括父类的属性。

在一般开发中，如果实体类没有继承其他类，则可以将callSuper属性设置为false，这样可以减少生成的代码量。但是，如果实体类继承了其他类，而且需要在子类中重写父类的equals和hashCode方法，则应该将callSuper属性设置为true，以包括父类的属性。

callSuper属性的取值应该根据实际情况来决定，需要根据实体类是否继承其他类来选择合适的取值。

# @Accessors注解

@Accessors(chain = true)：这是Lombok框架中的注解，它用于配置链式调用。通过在类上添加@Accessors注解，并将chain属性设置为true，可以启用链式调用。在这里，它的作用是启用SysResource类的链式调用。

@Accessors(chain=true) chain属性默认为false

链式访问，该注解设置chain=true，生成setter方法返回this（==也就是返回的是对象==），代替了默认的返回void。

@Data
@Accessors(chain=true)
public class User {
    private Integer id;
    private String name;
    private Integer age;

    public static void main(String[] args) {
        //开起chain=true后可以使用链式的set
        User user=new User().setAge(31).setName("zhangsan");//返回对象
        System.out.println(user);
    }
}

# @RequiredArgsConstructor注解

当我们使用@Autowired进行Bean注入的时候，IDEA会提示警告，不建议使用此方式进行注入。Spring官方更推荐使用==构造方法==进行注入。

随之而来的问题就是如果一个类中要注入多个对象的话，构造方法进行注入的方式会显得代码很臃肿

这是 Lombok提供的一种注解，它会生成一个包含所有已经声明的字段的构造函数。如果某个字段被 final 修饰，那么它将会被包含在构造函数中。这个注解可以简化代码，并且使得创建对象的过程更加容易。需要注意的是，该注解仅作用于类中使用了 @NonNull 注解的字段。

public class UserController extends DcyBaseController {
	//这里使用的是final,若不使用final,用@NotNull注解也是可以的
    private final UserInfoService userInfoService;
    //	......
    
    // 标记为final的,会自动进行注入
    private final AliaPay aliaPay;
	
	// 使用lombok的@NonNull注解标记的,会自动进行注入
    @NonNull
    private JingDongPay jingDongPay;
	
	// 未标记final或@NonNull,不会进行注入
    private WeixinPay weixinPay;
    
}
