# @ApiSupport注解

@ApiSupport注解用于对接口进行排序，其中order属性表示排序的优先级，数字越小表示越靠前。该注解能够让我们方便地控制各接口在Swagger文档中的展示顺序，提高了接口文档的可读性和易用性。

# @Api注解

@Api注解则用于定义Swagger文档的基本信息，包括接口名称、标签、描述等信息。其中，value属性表示接口名称，tags属性表示接口标签。使用@Api注解可以让Swagger生成更加规范和易懂的接口文档，帮助开发人员更好地理解和使用接口。

# @ApiOperation注解

@ApiOperation注解用于定义一个操作，其value属性表示操作的简要说明，notes属性表示操作的详细说明。该注解可以让开发人员在Swagger文档中快速了解接口的作用、参数、返回值等信息，提高接口的可读性。

# @ApiOperationSupport注解

@ApiOperationSupport注解则用于支持ApiOperation注解，其中order属性用于对接口进行排序，author属性用于指定接口作者。使用@ApiOperationSupport注解可以更好地控制接口在Swagger文档中的排序和作者信息，提高接口文档的可读性和易用性。

# @ApiParam注解

@ApiParam注解用于对接口参数进行描述。在控制器方法中使用该注解，可以为接口中的参数添加说明、示例值等元数据信息，进而生成更加详细和易读的接口文档。

在这个示例代码中，@ApiParam注解应用于方法参数loginInVO上面，表示该参数需要被记录到Swagger接口文档中，并为该参数添加说明。

# JavaBean中使用

@Data
@ApiModel
public class LoginInVO {

    @ApiModelProperty(value = "用户名")
    @NotBlank(message = "用户名不能为空")
    private String username;

    @ApiModelProperty(value = "密码")
    @NotBlank(message = "密码不能为空")
    private String password;
}

# @ApiModel注解

@ApiModel注解是Swagger中提供的用于对类进行描述的注解，在控制器中使用时可以生成更加详细的接口文档。在这个示例代码中，@ApiModel注解应用于类LoginInVO上面，表示该类为一个值对象，并对其进行了描述。

# @ApiModelProperty注解

@ApiModelProperty注解也是Swagger提供的注解之一，用于对模型属性进行描述。在这个示例代码中，@ApiModelProperty注解应用于类属性username和password上面，表示它们分别对应用户名和密码，并指定了对应属性的含义和说明信息。
