# 前后端对象的(反)序列化

序列化

如果你想在前端传递一个对象到后端，通常需要将对象序列化为字符串后再传递。

前端	->	对象	->	序列化	->	字符串	->	后端
1

如果你想将后端的 Java 对象传递给前端，通常需要将对象序列化为字符串后再传递。

后端	->	对象	->	序列化	->	字符串	->	前端
1

也就是说，前后端对象的传输通常都是通过 JSON 字符串来实现的

反序列化

我们这里就只考虑后端的反序列化，后端Spring中默认使用Jackson将 JSON 字符串反序列化为 Java 对象

前端	->	字符串	->	后端	->	反序列化	->	对象
1

Spring中一般我们都是通过注解实现序列化和反序列化

例如 @RequestBody 注解用于反序列化请求体，而 @RestController 注解用于将方法返回的对象序列化为响应体。

序列化中存在的问题

有个地方需要注意的是，Jackson会自动将字段名的首字母转化为小写，例如 XAxis 我们表示 X轴字段

但是在序列化或变成了 xaxis

例如，

@Data
@AllArgsConstructor
@NoArgsConstructor
public class User implements Serializable {
	private static final long serialVersionUID = 1L;
	private Integer id;
	private String name;
	private Integer age;
	private String[] XAxis;   //  x轴
}

解决办法：在 XAxis 字段上面添加@JsonProperty("XAxis")即可解决序列化问题

@Data
@AllArgsConstructor
@NoArgsConstructor
public class User implements Serializable {
    private static final long serialVersionUID = 1L;
    private Integer id;
    private String name;
    private Integer age;
    @JsonProperty("XAxis")
    private String[] XAxis;	//	X轴
}
