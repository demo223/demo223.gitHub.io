首先，org.mybatis.spring.boot.mybaits-spring-boot-starter 是 MyBatis 官方提供的 Spring Boot Starter，它整合了 MyBatis 和 Spring Boot，并提供了自动配置和注解支持，可以大大简化我们的配置工作。在使用这个 Starter 的时候，我们只需要在 pom.xml 中添加这个依赖，然后使用 @MapperScan 注解来扫描 Mapper 接口，就可以自动完成 MyBatis 的配置和集成。

其次，org.mybatis.mybatis 则是 MyBatis 核心依赖库。如果你希望手动配置 MyBatis，或者使用单独的 MyBatis 框架搭建项目，那么需要使用这个库。在使用 mybatis 依赖时，你需要手动配置一些 MyBatis 的核心组件，如 SqlSessionFactory、MapperScannerConfigurer 等等。使用起来可能比较麻烦，但也更加灵活。

总的来说，如果你是在 Spring Boot 中使用 MyBatis，推荐使用官方提供的 Starter org.mybatis.spring.boot.mybaits-spring-boot-starter。这个 Starter 已经为我们提供了大量的自动配置和注解支持，可以让我们快速地搭建一个 MyBatis + Spring Boot 项目。

如果你想要手动配置 MyBatis，或者使用单独的 MyBatis 框架，那么推荐使用 mybatis 依赖，并手动配置所需的组件。

Lombok提供了一组注解来简化Java代码，以下是一些常用的注解：

  1. @NoArgsConstructor, @RequiredArgsConstructor 和 @AllArgsConstructor：用于生成类的构造器
  2. @Data： 用于生成类的所有getter, setter, toString, equals, hashCode和构造器
  3. @Slf4j: 用于生成logger字段
  4. @SneakyThrows：用于抛出异常
  5. @Synchronized：用于生成同步方法

Apache commons-lang是Apache的一个开源项目，提供了一组Java常用类库。

其中，commons-lang3包含了一组Java工具类，提供了对字符串，数组，数字，日期等常用类型的操作方法。这些工具类大多数都是基于JDK中现有类的扩展，提供了更高级的功能和语法简洁的API。

例如，commons-lang3提供了StringUtils类，提供了一组字符串操作方法，如判断字符串是否为空，替换字符串，截取字符串等。

Apache commons-io是Apache的一个开源项目，提供了一组Java IO工具类库。

其中，commons-io包含了一组Java IO工具类，提供了对文件，目录，流，输入输出等常用类型的操作方法。这些工具类大多数都是基于JDK中现有类的扩展，提供了更高级的功能和语法简洁的API。

例如，commons-io提供了FileUtils类，提供了一组文件操作方法，如复制文件，读取文件内容，比较两个文件是否相同等。例如，使用FileUtils.copyFile(File srcFile, File destFile)方法来实现文件复制。

另外，commons-io还提供了一组流操作工具类，如IOUtils类，可以方便的读取输入流，写入输出流，关闭流等。

# SpringBoot起步依赖

Spring Boot提供了很多起步依赖，用于快速添加常用功能的依赖。

一些常用的起步依赖如下：

  • spring-boot-starter-parent：用来提供相关的Maven默认依赖，常用的包依赖可以省去version标签

  • spring-boot-starter-web：提供web开发所需的依赖，包括Spring MVC和Tomcat等

  • spring-boot-starter-data-jpa：提供整合Spring Data JPA所需的依赖

  • spring-boot-starter-security：提供Spring Security所需的依赖

  • spring-boot-starter-test：提供测试所需的依赖，包括JUnit和Mockito等

  • druid-spring-boot-starter：提供druid的数据源，数据库连接池等

  • spring-boot-starter-actuator：提供actuator监控所需的依赖

  • spring-boot-starter-data-redis：提供整合Redis所需的依赖

  • spring-boot-starter-aop：提供整合AOP所需的依赖
